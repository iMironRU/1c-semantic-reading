# § 2.2. Типы значений: число, строка, булево, дата

В прошлом параграфе мы установили: у программы есть форма и есть смысл, и работать со смыслом — отдельный навык. Тогда же мы заметили, что одинаковые по форме значения — `5` и `"5"` — на деле относятся к разным «сортам данных» и ведут себя по-разному. Об этих сортах — настоящий разговор.

В этом параграфе мы систематически разберёмся с *типами значений* в 1С: какие типы вообще бывают, чем они отличаются, какие операции для них допустимы, что происходит, когда они встречаются друг с другом. Это самый «технический» параграф Модуля 2, и он же — самый важный. Большинство сложностей, с которыми вы будете сталкиваться при чтении и написании кода, имеют корни здесь.

## 1. Что такое тип значения

Возьмём знакомый список:

```
Цена = 250;
Имя = "Иван";
ЭтоВыходной = Истина;
ДатаПродажи = '20251207';
```

Все четыре строки — присваивания. Левая часть везде одинакова по структуре (имя переменной), знак тоже один и тот же. А правые части — *совершенно разные*. Каждая принадлежит своему «сорту»:

- `250` — число.
- `"Иван"` — строка (текст в кавычках).
- `Истина` — логическое значение (Да/Нет).
- `'20251207'` — дата (записана в особых кавычках-апострофах; разберём её позже).

Этот «сорт» каждого значения и называется его **типом**. Тип определяет, *что это за значение* и *что с ним можно делать*. Над числом можно выполнять арифметику, к строке — приклеивать другие строки, логическое значение — проверять в условии, у даты — узнавать год или день недели.

> **Тип — это «сорт» значения, определяющий, что это за значение и какие операции с ним допустимы.**

Если выразить ту же мысль строже — *тип это внутреннее свойство значения, по которому платформа решает, как с этим значением обращаться: какие операции допускать, как сравнивать с другими значениями, как представлять при выводе*.

В платформе 1С каждое значение в каждый момент имеет некоторый тип. Значение `250` — это число; значение `"Иван"` — строка. Тип — *не пометка снаружи*, а внутренняя характеристика самого значения.

При этом *тип переменной* не задаётся раз и навсегда. У переменной нет «своего» типа, который объявляется заранее. Тип переменной в каждый момент — это тип того значения, которое в ней лежит сейчас. Если присвоить переменной новое значение другого типа, тип переменной соответственно изменится:

```
Х = 250;        // сейчас Х — число
Х = "Иван";     // теперь Х — строка
Х = Истина;     // теперь Х — булево
```

Это особенность 1С: переменная «сама по себе» бестипна, и приобретает тип от своего текущего содержимого. Во многих других языках программирования это устроено иначе — там тип переменной задаётся явно при её создании. Запомните это различие, чтобы потом не путаться.

Когда платформа доходит до строки кода с операцией, она *проверяет типы* участников: можно ли тут выполнить эту операцию? Если можно — выполняет. Если нет — выдаёт ошибку. Эта проверка — одна из главных причин, почему семантика так связана с типами.

## 2. Число

Самый знакомый и самый удобный тип. **Число** в 1С — это любое числовое значение: целое, дробное, положительное, отрицательное.

```
Цена = 250;
Расстояние = 12.5;
Температура = -3;
Эпсилон = 0.0001;
```

Все четыре переменные содержат числа. Заметьте: в коде дробная часть отделяется *точкой*, а не запятой (`12.5`, не `12,5`). Это техническая особенность, унаследованная из английской записи чисел. На экран при выводе число может попасть и с запятой, и с точкой — это зависит от настроек и места применения, — но *в самом коде* всегда точка.

С числами можно делать обычную арифметику: складывать (`+`), вычитать (`-`), умножать (`*`), делить (`/`). Это мы хорошо знаем из § 1.2. Можно также сравнивать: больше, меньше, равно, не равно (§ 1.4). Результат сравнения — уже не число, а логическое значение.

```
А = 5 + 3        →  А = 8           (число)
Б = 10 / 4       →  Б = 2.5         (число с дробной частью)
В = 5 > 3        →  В = Истина      (логическое значение)
```

В 1С нет жёсткого деления между «целыми» и «дробными» числами — они все одного типа. Деление `10 / 4` спокойно даёт `2.5`, и это не «другая разновидность числа», это просто число. В этом 1С отличается от многих других языков (Python, C, Java), где целые и дробные числа разделены жёстко. Запомните эту особенность — она нам ещё пригодится.

## 3. Строка

**Строка** — это последовательность символов: букв, цифр, знаков, пробелов. Записывается в *двойных кавычках*:

```
Имя = "Иван";
Адрес = "ул. Ленина, дом 5";
ПустоеМесто = " ";
Ничего = "";
```

Кавычки — обязательная часть записи. Без кавычек платформа поймёт `Иван` как имя переменной (которой, скорее всего, не существует). С кавычками — как готовое значение типа «строка», содержащее буквы И, в, а, н.

Заметьте важный момент: внутри кавычек могут быть *любые* символы — в том числе цифры. И тогда возникает ловушка, которую мы уже видели:

```
А = 5         (число)
Б = "5"       (строка из одной цифры)
```

Внешне очень похожи, а это совершенно разные значения. С `А` можно делать арифметику; с `Б` — нет. С `Б` можно делать «строковые» операции (склеивать с другой строкой, измерять длину); с `А` — нет.

Главная операция со строками — *склеивание* (конкатенация), знак `+`:

```
Имя = "Иван";
Фамилия = "Петров";
Полное = Имя + " " + Фамилия;     // "Иван Петров"
```

Кроме склеивания, со строками можно делать *сравнение*. `"Иван" = "Иван"` даёт `Истина`; `"Иван" = "Анна"` — `Ложь`. Сравнения «больше/меньше» для строк тоже работают: они сравнивают строки по алфавиту, но это редкая операция, и углубляться в неё мы не будем.

В платформе есть много встроенных команд для работы со строками — взять часть, найти подстроку, перевести в верхний регистр и так далее. С ними мы будем встречаться постепенно, по мере того как они понадобятся.

## 4. Логическое значение (булево)

С логическими значениями мы уже подробно знакомились в § 1.4. Повторим коротко.

**Логическое значение** — это значение, которое отвечает на вопрос «да или нет». Возможных значений всего два:

```
Найден = Истина;
ОшибкаЕсть = Ложь;
```

Слово «булево» в названии раздела происходит от фамилии английского математика Джорджа Буля, который в XIX веке создал алгебру логики. С тех пор два значения «истина» и «ложь» в программировании называют *булевыми*, и тип значения иногда так и называют — **Булево** (это название использует и сама платформа 1С).

Откуда берутся логические значения? В подавляющем большинстве случаев — из *сравнений*. Любое выражение со знаком сравнения (`>`, `<`, `=`, `<>` и так далее) даёт логическое значение:

```
А = 5 > 3            →  А = Истина
Б = "Иван" = "Анна"  →  Б = Ложь
В = Цена < 100       →  В = Истина или Ложь, зависит от Цены
```

Реже значения `Истина` и `Ложь` пишут в коде напрямую — обычно когда нужно явно выразить начальное состояние или флаг:

```
ЭтоПервыйЗапуск = Истина;
ОшибкаОбнаружена = Ложь;
ЭтоПостоянныйКлиент = Истина;
```

Главное применение логических значений — в условии `Если`. Условие должно вычислиться в логическое значение; если `Истина` — тело выполняется, если `Ложь` — пропускается.

Логические значения — не «уменьшенные числа» и не «короткие строки». Это самостоятельный тип со *своей* арифметикой. Складывать или умножать их как числа — некорректно: для них есть свои операции, и они есть главное продолжение этого раздела.

## 5. Логические операции: И, ИЛИ, НЕ

В реальных программах редко обходятся одним простым условием вроде `Цена > 1000`. Гораздо чаще условие *составное*: «если клиент постоянный *и* сумма больше тысячи», «если документ проведён *или* отменён», «если пользователь *не* администратор». Слова «и», «или», «не» — это не литературные связки, а полноценные *логические операторы*, и в 1С они записываются именно так:

```
И     — оба условия должны быть истинны
ИЛИ   — достаточно, чтобы было истинно хотя бы одно
НЕ    — переворачивает значение: Истина в Ложь, Ложь в Истина
```

Все три оператора работают над логическими значениями и дают в результате логическое значение. Самое наглядное — посмотреть, как они себя ведут на всех возможных входах:

```
А         Б         А И Б      А ИЛИ Б
Истина    Истина    Истина     Истина
Истина    Ложь      Ложь       Истина
Ложь      Истина    Ложь       Истина
Ложь      Ложь      Ложь       Ложь
```

Для `НЕ` ещё проще:

```
А         НЕ А
Истина    Ложь
Ложь      Истина
```

В реальном коде эти операторы появляются почти всегда внутри условия `Если`. Пример из «Канцтоваров»:

```
// Скидка действует только в выходные постоянным клиентам
Если ЭтоПостоянныйКлиент И ЭтоВыходной Тогда
    ПроцентСкидки = 10;
КонецЕсли;

// Уведомить, если на складе пусто или почти пусто
Если ОстатокНаСкладе = 0 ИЛИ ОстатокНаСкладе < 5 Тогда
    Сообщить("Срочно заказать!");
КонецЕсли;

// Не пускать неавторизованных
Если НЕ ПользовательАвторизован Тогда
    Сообщить("Требуется вход в систему.");
КонецЕсли;
```

> **Логические операторы `И`, `ИЛИ`, `НЕ` нужны для составления составных условий. Каждый принимает логические значения и возвращает логическое значение.**

**Приоритет операторов.** Когда в одном выражении встречаются разные операторы — арифметические, сравнения и логические, — платформа применяет их в определённом порядке:

```
1. Арифметические: *, /, +, -    (сначала)
2. Сравнения: =, <>, <, >, <=, >=
3. НЕ
4. И
5. ИЛИ                            (в последнюю очередь)
```

Например:

```
Если Цена > 1000 И Количество < 5 Тогда
```

Платформа сначала вычислит `Цена > 1000` (получит Истина или Ложь), потом `Количество < 5` (тоже Истина или Ложь), и только потом применит к ним `И`. Никаких скобок не нужно — приоритет всё расставит сам.

Но это не значит, что скобок надо избегать. *Если сомневаетесь — ставьте скобки* (правило, которое мы уже знаем по § 1.2). С логическими условиями оно работает так же: лишние скобки не повредят, а читаемость улучшат:

```
Если (Цена > 1000) И (Количество < 5) Тогда
```

Это *эквивалентный* код, но читать его глазам легче — границы между подвыражениями явны.

**Когда условий три и больше**, скобки иногда становятся не лишними, а *необходимыми*. Сравните:

```
А ИЛИ Б И В         // Сначала Б И В, потом ИЛИ
(А ИЛИ Б) И В       // Сначала А ИЛИ Б, потом И с В
```

Это разные выражения, и при некоторых значениях входов они дают разные результаты. Если хотите, чтобы поведение было однозначным и легко читалось — берите в скобки то, что должно вычисляться первым.

**Чего мы не делаем в этом курсе.** В математической логике вокруг этих трёх операций построена целая ветвь — *булева алгебра*. Там есть законы преобразования (де Моргана, дистрибутивность, поглощение и так далее), и для глубокого программирования они полезны. Но это отдельная большая тема, для которой нужен отдельный курс. Нам сейчас хватит того, что написано выше: понимать, что делает каждый оператор, уметь применять их в условиях, аккуратно расставлять скобки.

## 6. Дата

Это единственный новый тип в этом параграфе. Со встроенными датами мы ещё не работали.

**Дата** в 1С — это значение, представляющее собой *момент во времени*: конкретную дату, или дату со временем, или просто время. В коде дата записывается в *одинарных кавычках* (апострофах) в формате `'ГГГГММДД'` или `'ГГГГММДДЧЧММСС'`:

```
ДатаРождения = '19850315';               // 15 марта 1985 года
ДатаПроведения = '20251207120000';       // 7 декабря 2025 года, 12:00:00
СегодняВПолдень = '20251207120000';      // то же самое, иначе записанное
```

Зачем в дате *одинарные* кавычки, а в строке *двойные*? Чтобы платформа сразу понимала, что это за значение: то и другое — текст в кавычках, но разница в типе кавычек делает первое строкой, второе датой. Это не косметическое различие. Внешне `"20251207"` и `'20251207'` похожи, но *типы у них разные*: первое — обычная строка из восьми цифр; второе — дата 7 декабря 2025 года. Платформа отличает их буквально: один символ кавычки — и тип уже другой. Это снова та самая буквальность из § 0.4: один лишний или пропущенный знак — другой тип, другое поведение.

Заметьте необычность формата. В 1С дата в коде *всегда* записывается слитно: `20251207`, а не `2025-12-07` или `07.12.2025`. Год идёт первым (это четыре цифры), потом месяц (две цифры), потом день (две цифры). Если есть время — далее часы, минуты, секунды. Если время не указано — считается «начало суток» (полночь).

Этот формат — особенность 1С. В других языках программирования даты записываются по-другому: иногда строкой, иногда числом, иногда специальной командой. Здесь — апострофы и слитный год-месяц-день.

С датами можно делать осмысленные вещи. Самое простое:

```
Сообщить(ДатаРождения)        →  выведется в формате, понятном пользователю,
                                  например «15.03.1985»
```

Можно сравнивать даты — какая раньше, какая позже:

```
ДатаНачала = '20250101';
ДатаКонца = '20251231';
ВПорядкеЛиЗадан = ДатаНачала < ДатаКонца;   // Истина
```

Можно вычитать одну дату из другой — получится разница *в секундах* (числом). Можно прибавлять к дате число секунд — получится новая дата (на столько-то секунд вперёд):

```
ДатаНачала = '20250101';
ДатаКонца = '20250102';
РазницаВСекундах = ДатаКонца - ДатаНачала;
// РазницаВСекундах = 86400 (число секунд в одних сутках)
```

Заметьте: *результат вычитания двух дат — это уже не дата, а число*. Типы участников «изменились» в результате операции. С этим всегда нужно быть внимательным: тип результата может отличаться от типов входов.

В платформе есть команды-помощники для работы с датами: получить только дату из «даты со временем», узнать день недели, добавить календарный месяц. Этим мы займёмся в дальнейшем; сейчас нам важно одно — *дата это отдельный тип*, со своими правилами, и к этим правилам надо относиться внимательно.

> **Дата — это значение особого типа: момент во времени. В коде записывается в одинарных кавычках в формате `'ГГГГММДД'` или `'ГГГГММДДЧЧММСС'`.**

## 7. Когда типы встречаются

До сих пор мы рассматривали типы по отдельности. На практике они постоянно встречаются друг с другом — в одном выражении, в условии, в команде вывода. Что происходит, когда типы смешиваются?

Здесь есть три случая.

**Случай первый — операция определена для обоих типов.** Например, сравнение `=` работает и для чисел, и для строк, и для дат, и для логических значений (при условии, что сравниваются значения одного типа). Платформа просто выполняет сравнение, как мы и ожидаем.

**Случай второй — типы участников допускают «приведение друг к другу».** Самый распространённый пример — оператор `+` со строкой и числом. Платформа умеет «превратить» число в его текстовое представление и склеить со строкой. Это мы уже видели в § 1.2 и § 2.1:

```
Сообщить("Цена: " + 100)        →  "Цена: 100"
```

Платформа фактически сделала так:

```
"Цена: " + 100   →   "Цена: " + "100"   →   "Цена: 100"
```

Число `100` неявно превратилось в строку `"100"`, и дальше работа пошла как обычная конкатенация. Это называется *неявным приведением типа*: «неявным» — потому что мы об этом нигде не просили, платформа сделала сама.

Здесь самое время понять, *почему* знак `+` ведёт себя по-разному. Дело в том, что в 1С (как и во многих других языках) у оператора `+` *два разных смысла*: для чисел это сложение, для строк — склеивание. Один и тот же знак, два разных действия. Какое именно действие выполнится, платформа решает *по типам участников*: если оба числа — складывает; если хотя бы один из участников строка — склеивает (а число при этом превращает в текст). Такое явление, когда один знак значит разное в зависимости от типов операндов, называется *перегрузкой оператора*. Слово «перегрузка» здесь не в плохом смысле «перегружено лишним», а в техническом — «один знак нагружен несколькими ролями».

То же происходит со склеиванием строки и логического значения (как мы видели в § 1.4):

```
"Скидка нулевая? " + (Скидка = 0)    →   "Скидка нулевая? Да" (или "Нет")
```

И со склеиванием строки и даты:

```
"Дата рождения: " + '19850315'       →   "Дата рождения: 15.03.1985"
```

Во всех этих случаях платформа автоматически переводит «не-строку» в её текстовое представление и склеивает со строкой.

Текстовое представление, которое платформа выбирает по умолчанию, не всегда нас устраивает. Число `1662.5` выведется именно как `1662.5` — а нам, скажем, хотелось бы видеть в чеке `1 662,50 руб.`. Дата `'19850315'` выведется как `15.03.1985` — а нам, может быть, нужно `15 марта 1985 г.`. Для управления тем, как значение превращается в текст, в платформе есть встроенная функция `Формат()`. Запись `Формат(1662.5, "ЧДЦ=2; ЧРГ=' '")` даст строку `"1 662,50"`. Сейчас углубляться в правила форматирования не будем — мы вернёмся к этому в Модуле 5, когда будем разбирать обороты реального кода. Запомнить надо одно: *одно и то же значение может быть представлено в виде текста по-разному*, и `Формат()` — главный инструмент управления этим.

**Случай третий — операция для участников не определена.** Платформа выдаёт *ошибку*. Например:

```
"Иван" - 5      →  ошибка: вычесть число из строки нельзя.
"5" * 3         →  ошибка: умножить строку на число нельзя.
'20250101' * 2  →  ошибка: умножить дату на число нельзя.
```

Помните упражнение 5 из § 1.3? Там программа спотыкалась именно на третьем случае: к концу выражения слева накопилась длинная строка, а справа стояло число с операцией вычитания. Вычесть число из строки нельзя — платформа остановилась с ошибкой. Теперь это поведение становится понятным: типы участников не дают платформе выполнить операцию, и она честно сигнализирует.

Запомните общую картину:

```
Числа       арифметика, сравнения
Строки      склеивание, сравнения
Логические  логические операции (И, ИЛИ, НЕ), проверка в Если, сравнения
Даты        сравнения, специальная арифметика (разность дат, сдвиги)

Числа + Строки      → склеивание со строкой (число превращается в текст)
Строки + Логические → склеивание со строкой
Строки + Даты       → склеивание со строкой
Все остальные смеси → ошибка или специальные правила
```

## 8. Один особо коварный случай

Со смешением типов есть одна хитрость, на которой постоянно спотыкаются начинающие. Помните пример из § 2.1?

```
Скидка = "10"
Сообщить(Скидка + 5)     →  "105"
```

Здесь нет никакой ошибки. Программа не остановилась, не выдала никакой претензии. Она спокойно выполнила склеивание (потому что слева строка) и вывела `"105"`. Но *смысл* у этого результата нет: пользователь, который читал такое сообщение, ожидал бы увидеть `15` (десять плюс пять), а увидел `105`.

Это и есть самый коварный сценарий типов: программа работает, но делает не то, что задумано. И ошибка только одна — *в одном символе*: лишние кавычки вокруг `10`. Если бы было `Скидка = 10` (без кавычек), программа честно выполнила бы арифметику и выдала `15`. С кавычками — пошла другой дорогой.

Из этого следует практический совет, который полезно запомнить с самого начала:

> **Когда работаете с числами — пишите числа без кавычек. Когда работаете со строками — пишите в кавычках. Перепутывать эти случаи — самый частый источник «непонятных» ошибок.**

Внимательно смотрите на кавычки. Один-единственный лишний или пропущенный символ кавычки меняет тип значения, и через это меняется поведение всей программы.

## 9. Как узнать тип значения

В реальном коде иногда бывает непросто определить, какого типа окажется значение в переменной. Особенно если эта переменная пришла откуда-то «снаружи»: от пользователя, из документа, из настроек.

Для таких случаев в платформе есть команды-помощники:

- `ТипЗнч(Значение)` возвращает информацию о типе значения. Например, `ТипЗнч(5)` — это что-то про «Число», `ТипЗнч("5")` — про «Строку».
- Можно сравнить тип со специальной отметкой типа: `ТипЗнч(Х) = Тип("Число")` даст `Истина`, если `Х` — число.

Сейчас углубляться в эти команды не нужно — достаточно знать, что они есть. Когда понадобится — найдём их в синтакс-помощнике (мы говорили о нём в § 1.3) и применим.

## 10. Другие типы, которые мы встретим позже

Четыре типа — число, строка, булево, дата — это базовые встроенные типы платформы. Но их далеко не все. В 1С есть много других типов, которые мы будем встречать постепенно:

- **Ссылка на справочник** — «указатель» на конкретный элемент справочника товаров, клиентов, сотрудников. С ссылками мы будем разбираться в § 5.4.
- **Ссылка на документ** — то же самое для документов: ссылка на конкретный документ продажи, поступления, оплаты.
- **Структура** и **массив** — коллекции значений. Мы вскользь встретимся с массивами в § 3.4 и обстоятельно — в Модулях 4–5.
- **Неопределено** — особое значение «ничего», когда переменная ещё не получила содержимого или результат недоступен. Часто появляется при работе с базой данных.
- **NULL** — ещё одно особое значение «ничего», но с другим смыслом, чем «Неопределено». С ним мы встретимся при работе с запросами (§ 8.4).

Каждый из этих типов имеет своё назначение, свои правила и свои операции. Углубляться сейчас в них не нужно — упомянули, чтобы было понятно: мир типов в 1С больше, чем «число, строка, дата, булево», и его освоение — постепенная работа.

Нам же на ближайшее время хватит четырёх базовых.

## 11. Канцтовары: типы в действии

Закроем параграф осмысленным примером, в котором встречаются все четыре типа сразу. В магазине «Канцтовары» оформляется продажа: школа №7 (постоянный клиент) покупает 50 тетрадей по 35 рублей с 5%-ой скидкой. Программа должна сформировать строку чека.

```
// Данные о продаже
ДатаПродажи = '20251207';
КлиентНаименование = "Школа №7";
ЭтоПостоянныйКлиент = Истина;

ЦенаЗаШтуку = 35;
Количество = 50;

// Расчёт
Стоимость = ЦенаЗаШтуку * Количество;

Если ЭтоПостоянныйКлиент Тогда
    ПроцентСкидки = 5;
Иначе
    ПроцентСкидки = 0;
КонецЕсли;

СуммаСкидки = Стоимость * ПроцентСкидки / 100;
ИтогоКОплате = Стоимость - СуммаСкидки;

// Формирование чека
Сообщить("=== Чек ===");
Сообщить("Дата: " + ДатаПродажи);
Сообщить("Клиент: " + КлиентНаименование);
Сообщить("Сумма к оплате: " + ИтогоКОплате + " руб.");
```

Посмотрите на типы переменных:

```
ДатаПродажи           — Дата
КлиентНаименование    — Строка
ЭтоПостоянныйКлиент   — Булево
ЦенаЗаШтуку           — Число
Количество            — Число
Стоимость             — Число (результат умножения двух чисел)
ПроцентСкидки         — Число
СуммаСкидки           — Число
ИтогоКОплате          — Число
```

Все четыре базовых типа — на сцене. И посмотрите, как они работают в выражениях:

- `ЦенаЗаШтуку * Количество` — умножение двух чисел, обычная арифметика.
- `Если ЭтоПостоянныйКлиент Тогда` — проверка логического значения. Можно было бы написать `Если ЭтоПостоянныйКлиент = Истина Тогда`, но и без этого условие работает: само значение `Истина` или `Ложь` платформа берёт за результат.
- `"Дата: " + ДатаПродажи` — склеивание строки с датой. Платформа автоматически превращает дату в её текстовое представление.
- `"Клиент: " + КлиентНаименование` — склеивание двух строк, обычная конкатенация.
- `"Сумма к оплате: " + ИтогоКОплате + " руб."` — последовательное склеивание строки, числа, строки. Каждое склеивание срабатывает отдельно, число превращается в текст по дороге.

Это и есть нормальная программа на 1С: разные типы участвуют в разных операциях, и работают вместе слаженно — *если* программист понимает, что какого типа и какие операции с ним допустимы.

## Главное в одном абзаце

Тип значения — это его «сорт», который определяет, что это за значение и какие операции с ним допустимы. В 1С у каждого значения всегда есть свой тип, и платформа проверяет типы при каждой операции. Четыре базовых типа: *число* (любое числовое значение, без разделения на целые и дробные), *строка* (текст в двойных кавычках), *булево* (логическое — Истина или Ложь), *дата* (момент во времени, в одинарных кавычках, формат `'ГГГГММДД'`). Числа складывают, умножают, делят, сравнивают; строки склеивают и сравнивают; булевы значения проверяют в условиях и комбинируют логическими операторами `И`, `ИЛИ`, `НЕ`; даты сравнивают и работают со сдвигами во времени. Когда типы смешиваются, платформа либо выполняет операцию (например, склеивает строку с числом, превратив число в текст), либо выдаёт ошибку (например, при попытке вычесть число из строки). Знак `+` поэтому *перегружен*: для чисел это сложение, для строк — склеивание; что выберет платформа, решают типы операндов. Главная коварная ловушка — *запутаться между числом 5 и строкой "5"*: одно отличие в кавычках — и `5 + 3` превращается в `"53"` вместо `8`. Внимательно смотрите на кавычки. Значения преобразуются в текст по умолчанию довольно «грубо»; для красивого вывода (`1 662,50 руб.` вместо `1662.5`) есть встроенная функция `Формат()`. В курсе нам встретятся и другие типы — ссылки на справочники, документы, коллекции, особое значение Неопределено, — но их мы будем разбирать постепенно.

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

```
а)  100              →  Число
б)  "100"            →  Строка (содержит цифры, но в кавычках)
в)  Истина           →  Булево (логическое)
г)  -3.14            →  Число (отрицательное, дробное)
д)  ""               →  Строка (пустая, но всё равно строка)
е)  '20251207'       →  Дата
ж)  Ложь             →  Булево
з)  "Истина"         →  Строка (а не булево — из-за кавычек!)
и)  0                →  Число
к)  '20251207120000' →  Дата (с указанием времени)
```

Особо обратите внимание на пары: `Истина` (булево) и `"Истина"` (строка); `100` (число) и `"100"` (строка). Кавычки кардинально меняют тип. Это чисто техническая деталь, но она приводит к самым разным последствиям при выполнении программы.

**К упражнению 2.** *Полное решение.*

Прогоним по очереди:

- `А = 5 + 3` — сложение двух чисел: `А = 8` (число). Выведется: `8`.
- `Б = "5" + "3"` — склеивание двух строк: `Б = "53"` (строка из двух цифр). Выведется: `53` (но это уже строка, не число!).
- `В = 5 + "3"` — смешанная операция. Здесь работает правило приведения: при встрече числа и строки знак `+` означает склеивание, число превращается в строку. `В = "53"` (строка). Выведется: `53`.
- `Г = "Сумма: " + 100` — склеивание строки с числом: `Г = "Сумма: 100"` (строка). Выведется: `Сумма: 100`.
- `Д = Истина` — присвоили булево значение. Выведется: `Да`.
- `Е = "Ответ: " + Истина` — склеивание строки с булевым: булево превращается в текст «Да» (или «Нет»). `Е = "Ответ: Да"`. Выведется: `Ответ: Да`.

Заметьте: на экран в случаях Б и В попадает одна и та же надпись `53`, но *семантически* это разные значения. Б — результат склеивания двух строк-цифр. В — результат смешанной операции с приведением. На вид одинаково, по смыслу разное.

**К упражнению 3.** *Полное решение.*

```
а)  100 + 50          корректно. Сложение чисел, результат: 150 (число).
б)  "100" + "50"      корректно. Склеивание строк, результат: "10050" (строка).
в)  100 + "50"        корректно. Число превращается в строку, склеивается. 
                      Результат: "10050" (строка).
г)  "Иван" - "Петр"   ошибка. Вычитание для строк не определено.
д)  "Иван" * 3        ошибка. Умножение строки на число не определено.
е)  '20251207' < '20251231'   корректно. Сравнение двух дат, результат: 
                              Истина (булево). 7 декабря раньше 31-го.
ж)  Истина И Ложь     корректно. Логическая операция «И»: оба операнда 
                      должны быть истинны, но Ложь — не истина, поэтому 
                      результат — Ложь (булево).
з)  "Скидка " + 5 + "%"  корректно. Последовательно склеиваются строка, 
                         число (превратится в "5") и строка. Результат: 
                         "Скидка 5%" (строка).
и)  '20251231' - '20251207'   корректно. Вычитание дат — особая операция: 
                              результат не дата, а число секунд между ними. 
                              Здесь это 24 дня, то есть 24 × 86400 = 2 073 600 
                              секунд (число).
```

Запомните общее правило: операция `+` со строкой и числом — это всегда склеивание. Операция `-`, `*`, `/` со строкой и числом — это почти всегда ошибка. Сравнение работает только в пределах *одного типа*: число с числом, строку со строкой, дату с датой. Вычитание дат — особый случай: даёт число, а не дату.

**К упражнению 4.** *Полное решение.*

В строке `ВозрастКлиента = "25"` присвоено значение **строкового типа** — кавычки вокруг 25 делают это значение строкой, а не числом. На следующей строке `ВозрастКлиента >= 18` — сравнение строки с числом. Это разнотипное сравнение, и поведение платформы в таких случаях не должно служить опорой при разработке: даже если на одних входах оно сработает «как надо», на других — может повести себя неожиданно. Хорошее правило: *сравнение между значениями разных типов требует явного приведения*. Полагаться на то, что платформа «догадается», — ошибка проектирования.

Правильно — сразу присвоить число:

```
ВозрастКлиента = 25;
```

Без кавычек. Тогда тип переменной — число, сравнение с 18 прозрачно и однозначно.

Этот пример — реальный сюжет из практики. Часто разработчик копирует значение из какого-то текстового источника (например, из строкового поля формы) и забывает превратить его в число. Программа может какое-то время работать, потому что в простых случаях платформа выкручивается; но при первом же необычном случае (например, если в строке оказался пробел или запятая) — ломается. Такие баги ловятся только если с самого начала *не допускать* разнотипных сравнений.

**К упражнению 5.** *Полное решение.*

Возможный вариант программы:

```
// Данные о продаже
ДатаПродажи = '20251207'
ИмяКлиента = "Школа №7"
ЭтоПостоянныйКлиент = Истина
ЦенаЗаШтуку = 35
Количество = 50

// Расчёт
Стоимость = ЦенаЗаШтуку * Количество
Если ЭтоПостоянныйКлиент Тогда
    ПроцентСкидки = 5
Иначе
    ПроцентСкидки = 0
КонецЕсли;
СуммаСкидки = Стоимость * ПроцентСкидки / 100
ИтогоКОплате = Стоимость - СуммаСкидки

// Чек
Сообщить("=== Чек ===")
Сообщить("Дата: " + ДатаПродажи)
Сообщить("Клиент: " + ИмяКлиента)
Сообщить("Стоимость: " + Стоимость + " руб.")
Сообщить("Скидка: " + ПроцентСкидки + "%")
Сообщить("К оплате: " + ИтогоКОплате + " руб.")
```

Прогоним:

- `Стоимость = 35 * 50 = 1750`.
- `ЭтоПостоянныйКлиент = Истина`, поэтому `ПроцентСкидки = 5`.
- `СуммаСкидки = 1750 * 5 / 100 = 87.5`.
- `ИтогоКОплате = 1750 - 87.5 = 1662.5`.

В окне сообщений появится:

```
=== Чек ===
Дата: 07.12.2025
Клиент: Школа №7
Стоимость: 1750 руб.
Скидка: 5%
К оплате: 1662.5 руб.
```

В программе участвуют все четыре базовых типа: дата (`ДатаПродажи`), строка (`ИмяКлиента`), число (`ЦенаЗаШтуку` и многие другие), булево (`ЭтоПостоянныйКлиент`). Все они работают слаженно, потому что в каждом месте программист понимал, какого типа значения участвуют в операции, и какая операция уместна.
