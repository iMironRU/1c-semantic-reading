## § 3.3. Цикл «Для каждого Из»

В § 3.2 мы освоили цикл «Пока»: он повторяет тело, пока условие истинно, и требует от нас самостоятельно управлять счётчиком и следить за тем, чтобы цикл не стал бесконечным. Это мощный инструмент — но довольно трудоёмкий, когда задача простая: «пройди по всем элементам списка и сделай с каждым что-то».

Именно для таких задач в 1С есть второй вид цикла — **«Для каждого Из»**. Он не требует счётчика, не требует условия выхода и не может стать бесконечным по ошибке программиста. Он просто проходит по коллекции от первого элемента до последнего — и завершается сам.

---

## 1. Что такое коллекция

Прежде чем разбирать новый цикл, нужно ввести понятие, на котором он работает.

До сих пор наши переменные хранили *одно значение*: одно число, одну строку, одно логическое. Но в реальных задачах данные часто идут *группами*: список товаров, набор цен, перечень сотрудников. Хранить каждый элемент в отдельной переменной — неудобно и негибко.

**Коллекция** — это переменная, которая хранит *несколько значений* сразу, как упорядоченный список.

> **Коллекция — это группа значений, хранящихся вместе под одним именем.**

В 1С коллекций несколько видов. В этом параграфе мы познакомимся с самой простой — **массивом**. Остальные виды коллекций (список значений, таблица значений и другие) встретятся позже — в Модуле 5, когда будем разбирать 1С-специфичные конструкции.

---

## 2. Массив: создание и наполнение

Массив в 1С создаётся командой `Новый Массив`. Слово `Новый` здесь стоит мимоходом, без объяснения — вернёмся к нему в § 5.1, когда будем говорить об объектах. Пока воспринимайте `Новый Массив` как устойчивое выражение: «создать пустой список».

```
Товары = Новый Массив;
```

Теперь `Товары` — пустой массив. Чтобы добавить в него элементы, используем метод `.Добавить()`:

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Карандаш");
Товары.Добавить("Линейка");
Товары.Добавить("Тетрадь");
```

После этих четырёх команд массив `Товары` содержит четыре строки — в том порядке, в котором мы их добавили. Элементы нумеруются с нуля: первый — `Товары[0]`, второй — `Товары[1]`, и так далее. Но при работе с циклом «Для каждого Из» следить за номерами не нужно — цикл сам идёт по всем элементам по порядку.

---

## 3. Конструкция «Для каждого Из»: синтаксис и схема

Цикл «Для каждого Из» записывается так:

```
Для Каждого <Переменная> Из <Коллекция> Цикл
    <Тело>
КонецЦикла;
```

Здесь:
- `Для Каждого` — ключевые слова, открывающие конструкцию.
- `<Переменная>` — имя переменной, которая будет поочерёдно принимать значение каждого элемента коллекции. Имя выбираем сами.
- `Из` — ключевое слово-связка.
- `<Коллекция>` — переменная, хранящая коллекцию (в нашем случае — массив).
- `Цикл` — открывает тело. Без точки с запятой.
- `<Тело>` — команды, которые выполняются для каждого элемента.
- `КонецЦикла;` — закрывает тело. Точка с запятой обязательна.

Порядок выполнения:

```
  ┌──► берём следующий элемент коллекции
  │         │
  │    есть │ нет (коллекция закончилась)
  │    ещё  │     └──► выходим из цикла, идём дальше
  │         ▼
  └─── выполняем тело (Переменная = текущий элемент)
```

Цикл сам отслеживает, на каком элементе находится. Когда элементы заканчиваются — цикл завершается. Никакого счётчика, никакого условия.

> **Цикл «Для каждого Из» проходит по всем элементам коллекции по очереди — от первого до последнего — и завершается сам.**

---

## 4. Прогонка: таблица состояния

Посмотрим, как работает цикл на конкретном примере. Возьмём список товаров и выведем каждый на экран:

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Карандаш");
Товары.Добавить("Линейка");
Товары.Добавить("Тетрадь");

Для Каждого Товар Из Товары Цикл
    Сообщить(Товар);
КонецЦикла;
```

Прогоним. Переменная `Товар` поочерёдно принимает значение каждого элемента массива:

```
Итерация │ Товар        │ Вывод
─────────┼─────────────┼─────────────
    1    │ "Ручка"     │ Ручка
    2    │ "Карандаш"  │ Карандаш
    3    │ "Линейка"   │ Линейка
    4    │ "Тетрадь"   │ Тетрадь
    —    │ (элементов  │ —
         │  больше нет)│
```

На экран выведено четыре строки — по одной на каждый элемент. Цикл завершился, когда массив закончился.

Сравните с прогонкой «Пока» из § 3.2: там в таблице был столбец с условием и счётчик. Здесь ни того, ни другого — только текущий элемент и то, что с ним делаем.

---

## 5. Типичный приём: накопление по коллекции

«Для каждого Из» хорошо сочетается с накоплением — приёмом, который мы освоили в § 3.2. Теперь аккумулятор собирает значения не из счётчика, а из элементов коллекции.

Пример: в «Канцтоварах» есть прайс-лист с ценами. Нужно посчитать общую стоимость всех позиций.

```
Цены = Новый Массив;
Цены.Добавить(35);   // Ручка
Цены.Добавить(15);   // Карандаш
Цены.Добавить(25);   // Линейка
Цены.Добавить(60);   // Тетрадь

ОбщаяСтоимость = 0;

Для Каждого Цена Из Цены Цикл
    ОбщаяСтоимость = ОбщаяСтоимость + Цена;
КонецЦикла;

Сообщить("Итого: " + ОбщаяСтоимость + " руб.");
```

Прогон:

```
Итерация │ Цена │ ОбщаяСтоимость (после)
─────────┼──────┼───────────────────────
    1    │  35  │          35
    2    │  15  │          50
    3    │  25  │          75
    4    │  60  │         135
    —    │  —   │           —
```

На экран: `Итого: 135 руб.`

Переменная `ОбщаяСтоимость` начала с нуля и в каждой итерации прибавляла к себе очередную цену. Это то же накопление, что и в § 3.2, — но теперь источник значений не счётчик, а элементы массива.

---

## 6. «Пока» или «Для каждого Из»: когда что уместно

Два цикла, которые мы освоили, решают разные задачи. Вот простое правило:

| Задача | Цикл |
|--------|------|
| Есть коллекция, нужно обработать каждый элемент | `Для каждого Из` |
| Нужно повторить N раз | `Пока` со счётчиком |
| Нужно остановиться по внешнему условию | `Пока` |
| Нужно идти, пока хватает ресурса (денег, запаса) | `Пока` |

> **Если есть коллекция и нужно пройти по всем её элементам — «Для каждого Из» всегда предпочтительнее «Пока» со счётчиком: он короче, надёжнее и яснее выражает намерение.**

«Для каждого Из» нельзя написать неправильно в части завершения: цикл сам знает, когда коллекция закончилась. «Пока» — гибче, но требует аккуратности. Как только в программе появляется коллекция и задача «пройди по всем» — это сигнал, что нужен «Для каждого Из».

---

## Главное в одном абзаце

Цикл «Для каждого Из» предназначен для обхода коллекций — групп значений, хранящихся вместе под одним именем. Простейшая коллекция в 1С — *массив*: создаётся командой `Новый Массив`, наполняется методом `.Добавить()`. Конструкция `Для Каждого <Переменная> Из <Коллекция> Цикл ... КонецЦикла;` проходит по всем элементам по порядку: на каждой итерации переменная принимает значение очередного элемента, тело выполняется, затем цикл переходит к следующему. Когда элементы заканчиваются — цикл завершается сам, без условия и без счётчика. Это главное отличие от «Пока»: «Для каждого Из» освобождает программиста от управления порядком обхода. Приём накопления работает с этим циклом так же, как с «Пока»: аккумулятор инициализируется нулём до цикла и пополняется в каждой итерации. Правило выбора: если есть коллекция и нужно пройти по всем элементам — «Для каждого Из»; если нужно считать, останавливаться по условию или работать без готовой коллекции — «Пока».

## Упражнения

**Упражнение 1.** Прогоните цикл в голове и выпишите, что появится на экране.

```
Города = Новый Массив;
Города.Добавить("Москва");
Города.Добавить("Тула");
Города.Добавить("Орёл");

Для Каждого Город Из Города Цикл
    Сообщить("Доставка в: " + Город);
КонецЦикла;
```

---

**Упражнение 2.** Напишите программу для «Канцтоваров»: дан массив цен на товары. Найдите максимальную цену и выведите её на экран.

Массив для проверки:

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(15);
Цены.Добавить(80);
Цены.Добавить(25);
```

*Указание:* заведите переменную `Максимум` и инициализируйте её нулём. В каждой итерации проверяйте: если текущая цена больше `Максимум` — обновляйте `Максимум`.

---

**Упражнение 3.** Напишите программу, которая считает, сколько товаров в массиве стоят дороже 30 рублей. Используйте тот же массив цен из упражнения 2.

*Указание:* заведите счётчик, инициализируйте нулём. В теле цикла — ветвление: если цена > 30, увеличивайте счётчик.

---

**Упражнение 4.** Перед вами два варианта одной программы. Оба выводят товары из списка. Какой из них правильнее и почему?

*Вариант А:*

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Карандаш");
Товары.Добавить("Линейка");

Для Каждого Товар Из Товары Цикл
    Сообщить(Товар);
КонецЦикла;
```

*Вариант Б:*

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Карандаш");
Товары.Добавить("Линейка");

Счётчик = 0;
Пока Счётчик < 3 Цикл
    Сообщить(Товары[Счётчик]);
    Счётчик = Счётчик + 1;
КонецЦикла;
```

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

```
Доставка в: Москва
Доставка в: Тула
Доставка в: Орёл
```

Три итерации — по одной на каждый элемент массива. Переменная `Город` поочерёдно принимала значения `"Москва"`, `"Тула"`, `"Орёл"`. В теле каждый раз выполнялась одна команда: `Сообщить("Доставка в: " + Город)`.

---

**К упражнению 2.** *Полное решение.*

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(15);
Цены.Добавить(80);
Цены.Добавить(25);

Максимум = 0;

Для Каждого Цена Из Цены Цикл
    Если Цена > Максимум Тогда
        Максимум = Цена;
    КонецЕсли;
КонецЦикла;

Сообщить("Максимальная цена: " + Максимум + " руб.");
```

Прогон:

```
Итерация │ Цена │ Цена > Максимум │ Максимум (после)
─────────┼──────┼─────────────────┼─────────────────
    1    │  35  │ 35 > 0, Истина  │       35
    2    │  15  │ 15 > 35, Ложь   │       35
    3    │  80  │ 80 > 35, Истина │       80
    4    │  25  │ 25 > 80, Ложь   │       80
```

На экран: `Максимальная цена: 80 руб.`

Обратите внимание: начальное значение `Максимум = 0` работает корректно, потому что все цены положительны. Если бы цены могли быть отрицательными — начальное значение пришлось бы выбирать иначе.

---

**К упражнению 3.** *Полное решение.*

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(15);
Цены.Добавить(80);
Цены.Добавить(25);

КоличествоДорогих = 0;

Для Каждого Цена Из Цены Цикл
    Если Цена > 30 Тогда
        КоличествоДорогих = КоличествоДорогих + 1;
    КонецЕсли;
КонецЦикла;

Сообщить("Товаров дороже 30 руб.: " + КоличествоДорогих);
```

Из четырёх цен условию `> 30` удовлетворяют две: 35 и 80. На экран: `Товаров дороже 30 руб.: 2`

Здесь цикл и ветвление работают вместе: цикл обходит все элементы, ветвление решает, считать ли текущий. Это типичная комбинация — *фильтрация* коллекции по условию.

---

**К упражнению 4.** *Полное решение.*

Правильнее **Вариант А**.

Оба варианта выводят одно и то же. Но Вариант Б использует цикл «Пока» там, где есть готовая коллекция, — и тащит за собой весь груз ручного управления: счётчик, начальное значение, условие, шаг. Если завтра в массив добавят четвёртый товар — в Варианте Б нужно не забыть изменить `3` на `4`. В Варианте А — ничего менять не нужно: цикл сам определяет, сколько элементов.

Вариант А яснее выражает намерение: «пройди по всем товарам». Вариант Б выражает механику: «пройди три раза». Первое — про *что*, второе — про *как*. Когда есть коллекция, «Для каждого Из» — правильный выбор.

Теперь у нас есть оба кита повторения: «Пока» — для задач с условием, «Для каждого Из» — для обхода готовых наборов данных. В следующем § 3.4 займёмся третьим инструментом структурирования программы — процедурами и функциями: способом дать блоку команд имя и вызывать его столько раз, сколько нужно.*
