## § 3.4. Процедуры и функции

В предыдущих параграфах мы освоили все три кита структурного программирования: последовательность, ветвление и цикл. С их помощью можно написать любой алгоритм. Но по мере того как программа растёт, возникает новая проблема: одни и те же группы команд приходится писать снова и снова. Добавляется и другая: в длинном коде становится трудно понять, *что именно делает* тот или иной фрагмент.

Решение этих проблем — **подпрограммы**: процедуры и функции. Это способ дать блоку команд имя и вызывать его в любом месте программы столько раз, сколько нужно.

---

## 1. Зачем нужны подпрограммы

Представьте: в «Канцтоварах» нужно несколько раз напечатать разделительную линию между разделами чека. Без подпрограммы:

```
Сообщить("--------------------");
// ... пара команд ...
Сообщить("--------------------");
// ... ещё команды ...
Сообщить("--------------------");
```

Линия одна и та же, но написана трижды. Если завтра решат сделать её длиннее — придётся найти и исправить все три места. Пропустишь одно — чек будет некрасивым.

С подпрограммой:

```
Процедура НапечататьРазделитель()
    Сообщить("--------------------");
КонецПроцедуры;

НапечататьРазделитель();
// ... пара команд ...
НапечататьРазделитель();
// ... ещё команды ...
НапечататьРазделитель();
```

Теперь линия определена в одном месте. Хочешь изменить — меняешь один раз. А код стал читаться иначе: вместо строки символов — смысловое слово `НапечататьРазделитель`. Читатель понимает *что* происходит, не вникая в *как*.

> **Подпрограмма — это блок команд с именем. Определяется один раз, вызывается в любом месте программы столько раз, сколько нужно.**

---

## 2. Процедура: синтаксис

Процедура — подпрограмма, которая выполняет действия и **не возвращает** результат.

```
Процедура <Имя>(<Параметры>)
    <Тело>
КонецПроцедуры;
```

Пример без параметров:

```
Процедура НапечататьШапкуЧека()
    Сообщить("Магазин Канцтовары");
    Сообщить("--------------------");
КонецПроцедуры;
```

Вызов — просто имя со скобками:

```
НапечататьШапкуЧека();
```

При вызове выполнение переходит внутрь процедуры, проходит тело от начала до конца, а затем возвращается к следующей команде после вызова. Это точно такой же переход, как при входе в ветку `Если` — только адресуется по имени, а не по условию.

> **Вызов процедуры — это команда: «перейди туда, выполни, вернись».**

---

## 3. Функция: синтаксис и возврат значения

Функция — подпрограмма, которая выполняет действия и **возвращает** результат. Именно наличие возвращаемого значения отличает функцию от процедуры.

```
Функция <Имя>(<Параметры>)
    <Тело>
    Возврат <Значение>;
КонецФункции;
```

Ключевое слово `Возврат` — это команда: немедленно завершить выполнение функции и передать указанное значение туда, откуда её вызвали. Без `Возврат` функция не может передать результат — и в этом нет смысла писать её функцией, а не процедурой.

Пример:

```
Функция РассчитатьСкидку(Сумма, ПроцентСкидки)
    СуммаСкидки = Сумма * ПроцентСкидки / 100;
    Возврат СуммаСкидки;
КонецФункции;
```

Функцию вызывают там, где нужно значение — в присваивании или выражении:

```
ИтогоСкидка = РассчитатьСкидку(3500, 15);
Сообщить("Скидка: " + ИтогоСкидка + " руб.");
```

При вызове `РассчитатьСкидку(3500, 15)` выполнение переходит в тело функции, вычисляет `СуммаСкидки = 3500 * 15 / 100 = 525`, выполняет `Возврат 525` — и это значение `525` оказывается на месте вызова. Присваивание завершается: `ИтогоСкидка = 525`.

> **Функция отличается от процедуры тем, что возвращает значение. Это значение можно использовать в выражении или присвоить переменной.**

---

## 4. Параметры: первое знакомство

И процедура, и функция могут принимать **параметры** — переменные, объявленные в заголовке и получающие значение в момент вызова.

```
Функция РассчитатьСкидку(Сумма, ПроцентСкидки)
    // Сумма и ПроцентСкидки — параметры
    ...
КонецФункции;
```

При вызове `РассчитатьСкидку(3500, 15)` параметр `Сумма` получает значение `3500`, а `ПроцентСкидки` — значение `15`. Это происходит автоматически, по порядку: первый аргумент — первому параметру, второй — второму.

Параметры делают подпрограмму гибкой: одна и та же функция `РассчитатьСкидку` работает с любой суммой и любым процентом — не нужно писать отдельную функцию для каждого случая.

> **Параметры — это «дырки» в подпрограмме, через которые снаружи передаются значения. Они позволяют написать подпрограмму один раз и вызывать её с разными данными.**

Параметры — тема с тонкостями: способы передачи, область видимости, разница между пользовательскими и встроенными подпрограммами. Этому посвящён следующий § 3.5.

---

## 5. Встроенные и пользовательские подпрограммы

На протяжении всего курса мы вызываем `Сообщить(...)`, `Новый Массив`, обращаемся к `Товары.Добавить(...)`. Всё это — тоже подпрограммы. Но мы их не писали. Откуда они берутся?

В 1С подпрограммы бывают двух видов:

**Встроенные** — реализованы платформой. Их код нам недоступен; мы видим только имя и параметры. Именно к ним относятся `Сообщить`, `Формат`, `Лев`, `Найти` и сотни других функций платформы. Встроенные подпрограммы описаны в документации и синтакс-помощнике.

**Пользовательские** — пишем мы сами. Именно такие мы разбираем в этом параграфе: `НапечататьШапкуЧека`, `РассчитатьСкидку` и т. д.

```
  Встроенные               Пользовательские
  ──────────────────────   ──────────────────────
  Сообщить(...)            НапечататьШапкуЧека()
  Лев(Строка, 3)           РассчитатьСкидку(...)
  Формат(Число, "...")      ПроверитьОстаток(...)
  код скрыт платформой     код пишем мы
```

> **Встроенные подпрограммы реализованы платформой — их код нам недоступен, но интерфейс (имя и параметры) открыт. Пользовательские пишем мы сами.**

Встроенные подпрограммы имеют ещё одну особенность: некоторые из них принимают необязательные параметры. Например, `Сообщить` можно вызвать с одним или двумя аргументами. В пользовательских подпрограммах количество параметров всегда фиксировано — об этом подробнее в § 3.5.

---

## 6. Где живут подпрограммы: модули

В 1С пользовательские подпрограммы хранятся в **модулях** — специальных текстовых файлах с кодом. Модули привязаны к объектам конфигурации: есть модуль документа, модуль справочника, общие модули. Одна подпрограмма живёт в одном модуле.

Слово «модуль» здесь стоит мимоходом — полноценно разберём в § 8.2, когда будем говорить об архитектуре системы. Сейчас важно одно: подпрограммы не висят в воздухе, они организованы в файлы с кодом, и у каждой есть своё место.

---

## 7. Канцтовары: чек с подпрограммами

Соберём всё в развёрнутый пример. Программа печатает чек для покупателя «Канцтовары»:

```
Процедура НапечататьРазделитель()
    Сообщить("--------------------");
КонецПроцедуры;

Функция РассчитатьСкидку(Сумма, ПроцентСкидки)
    Возврат Сумма * ПроцентСкидки / 100;
КонецФункции;

Процедура НапечататьЧек(Товар, Цена, ПроцентСкидки)
    НапечататьРазделитель();
    Сообщить("Товар: " + Товар);
    Сообщить("Цена: " + Цена + " руб.");
    Скидка = РассчитатьСкидку(Цена, ПроцентСкидки);
    ИтогоКОплате = Цена - Скидка;
    Сообщить("Скидка: " + Скидка + " руб.");
    Сообщить("Итого: " + ИтогоКОплате + " руб.");
    НапечататьРазделитель();
КонецПроцедуры;

НапечататьЧек("Ручка", 35, 10);
НапечататьЧек("Тетрадь", 60, 5);
```

Прочитаем код, не прогоняя: видно три подпрограммы. `НапечататьРазделитель` — простейшая, без параметров. `РассчитатьСкидку` — вычислительная, возвращает число. `НапечататьЧек` — главная, вызывает обе внутри себя.

Два вызова в конце: один для ручки, другой для тетради. Код `НапечататьЧек` написан один раз, работает для любого товара.

На экран для первого вызова:

```
--------------------
Товар: Ручка
Цена: 35 руб.
Скидка: 3.5 руб.
Итого: 31.5 руб.
--------------------
```

Обратите внимание: `НапечататьЧек` вызывает `РассчитатьСкидку` внутри себя. Подпрограммы могут вызывать другие подпрограммы — это нормально и даже типично для реального кода.

---

## Главное в одном абзаце

Процедура и функция — это подпрограммы: блоки команд с именем, определённые один раз и вызываемые в любом месте программы. Процедура выполняет действия и не возвращает результат; функция отличается тем, что возвращает значение через ключевое слово `Возврат`. Оба вида принимают параметры — переменные, объявленные в заголовке и получающие значения при вызове; они делают подпрограмму гибкой и пригодной для повторного использования. В 1С подпрограммы бывают встроенными (реализованы платформой, код скрыт, интерфейс открыт) и пользовательскими (пишем мы сами и размещаем в модулях). Подпрограммы решают две задачи: устраняют повторы кода и дают фрагментам осмысленные имена — оба эффекта делают программу понятнее.

## Упражнения

**Упражнение 1.** Прочитайте код и ответьте: процедура это или функция? Что она делает? Что вернёт вызов `Результат = ВычислитьНДС(1000, 20)`?

```
Функция ВычислитьНДС(Сумма, СтавкаНДС)
    Возврат Сумма * СтавкаНДС / 100;
КонецФункции;
```

**Упражнение 2.** Найдите ошибку. Код должен напечатать приветствие, но что-то не так:

```
Процедура НапечататьПриветствие(Имя)
    Результат = "Добрый день, " + Имя + "!";
КонецПроцедуры;

Сообщение = НапечататьПриветствие("Александр");
Сообщить(Сообщение);
```

---

**Упражнение 3.** Напишите функцию `МаксимумДвух(А, Б)`, которая принимает два числа и возвращает большее. Проверьте для `МаксимумДвух(7, 12)` — должно вернуться `12`.

---

**Упражнение 4.** В магазине «Канцтовары» нужно напечатать итоговую строку для нескольких товаров. Напишите процедуру `НапечататьСтроку(Товар, Цена)`, которая выводит строку вида `Ручка — 35 руб.`, и вызовите её для трёх товаров: ручка (35 руб.), карандаш (15 руб.), тетрадь (60 руб.).

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Это **функция** — заканчивается на `КонецФункции` и содержит `Возврат`.

Она вычисляет сумму НДС: берёт сумму и ставку, возвращает их произведение, делённое на 100.

При вызове `ВычислитьНДС(1000, 20)`: параметр `Сумма` получает `1000`, `СтавкаНДС` — `20`. Вычисляется `1000 * 20 / 100 = 200`. Выполняется `Возврат 200`. Значение `200` оказывается на месте вызова. Присваивание: `Результат = 200`.

---

**К упражнению 2.** *Полное решение.*

Ошибка в том, что `НапечататьПриветствие` — это **процедура**, а не функция. Процедура не возвращает значений. Строка `Сообщение = НапечататьПриветствие("Александр")` пытается получить возвращаемое значение от процедуры — это некорректно.

Два варианта исправления:

*Вариант А — сделать функцию:*
```
Функция НапечататьПриветствие(Имя)
    Возврат "Добрый день, " + Имя + "!";
КонецФункции;

Сообщение = НапечататьПриветствие("Александр");
Сообщить(Сообщение);
```

*Вариант Б — вывести прямо в процедуре:*
```
Процедура НапечататьПриветствие(Имя)
    Сообщить("Добрый день, " + Имя + "!");
КонецПроцедуры;

НапечататьПриветствие("Александр");
```

Оба корректны. Выбор зависит от задачи: если нужно только вывести — процедура; если нужно получить строку и что-то с ней сделать — функция.

---

**К упражнению 3.** *Полное решение.*

```
Функция МаксимумДвух(А, Б)
    Если А >= Б Тогда
        Возврат А;
    Иначе
        Возврат Б;
    КонецЕсли;
КонецФункции;

Результат = МаксимумДвух(7, 12);
Сообщить(Результат);
```

Для `МаксимумДвух(7, 12)`: `А = 7`, `Б = 12`. Условие `7 >= 12` — Ложь. Идём в `Иначе`: `Возврат 12`. Результат: `12`.

Обратите внимание: в функции может быть несколько `Возврат` в разных ветках. Как только выполнение достигает любого из них — функция завершается с этим значением.

---

**К упражнению 4.** *Полное решение.*

```
Процедура НапечататьСтроку(Товар, Цена)
    Сообщить(Товар + " — " + Цена + " руб.");
КонецПроцедуры;

НапечататьСтроку("Ручка", 35);
НапечататьСтроку("Карандаш", 15);
НапечататьСтроку("Тетрадь", 60);
```

На экран:
```
Ручка — 35 руб.
Карандаш — 15 руб.
Тетрадь — 60 руб.
```

Три вызова, один код процедуры. Если завтра нужно изменить формат строки — меняем в одном месте, и все три строки обновятся автоматически.
