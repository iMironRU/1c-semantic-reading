## § 3.6. Блок-схемы как нотация

До сих пор мы записывали алгоритмы единственным доступным способом — кодом на 1С. Код точен, однозначен, выполняется компьютером. Но у него есть ограничение: чтобы понять алгоритм по коду, нужно уже знать язык. Есть и другой способ — **блок-схема**.

Блок-схема — это графическое представление алгоритма. Вместо слов и знаков препинания — фигуры и стрелки. Это не замена коду, а другой язык для той же идеи: алгоритм можно записать словами, кодом или рисунком. Каждая запись раскрывает разное.

---

## 1. Зачем блок-схемы

Блок-схема полезна в двух ситуациях.

**До написания кода.** Сначала рисуешь алгоритм — видишь всю структуру целиком, ловишь упущенные ветки и циклы. Потом переводишь в код. Это проще, чем сразу держать всю логику в голове и одновременно следить за синтаксисом.

**Для объяснения.** Блок-схема читается без знания языка программирования. Коллега, руководитель, заказчик — все поймут рисунок быстрее, чем листинг кода.

---

## 2. Базовые фигуры

В блок-схемах используют несколько фигур. Для нашего курса достаточно четырёх:

```
  ╭─────────╮
  │ НАЧАЛО  │  — овал (скруглённый прямоугольник)
  │  КОНЕЦ  │    Начало и конец алгоритма.
  ╰─────────╯

  ┌─────────────────┐
  │   Действие      │  — прямоугольник
  │   (команда)     │    Одна команда или группа команд.
  └─────────────────┘

  ◇─────────────────◇
  ◇   Условие?      ◇  — ромб
  ◇─────────────────◇    Проверка: выход «да» и «нет».

  ─────────────────────  — стрелка
                          Направление потока выполнения.
```

Правило: у каждого блока есть вход (стрелка сверху или сбоку) и выход (стрелка снизу или сбоку). Ромб — единственная фигура с двумя выходами.

---

## 3. Три конструкции в блок-схемах

Мы изучили три кита структурного программирования (§ 3.1). Вот как каждый выглядит в блок-схеме:

**Последовательность:**

```
    ╭────────╮
    │ НАЧАЛО │
    ╰───┬────╯
        │
    ┌───▼──────────┐
    │ Команда 1    │
    └───┬──────────┘
        │
    ┌───▼──────────┐
    │ Команда 2    │
    └───┬──────────┘
        │
    ╭───▼────╮
    │ КОНЕЦ  │
    ╰────────╯
```

**Ветвление (Если...Тогда...Иначе):**

```
        ┌──────────────┐
        │  Условие?    │
        └──────┬───────┘
               │
           ◇───▼───◇
      да ◇           ◇ нет
        ◇             ◇
       ◇               ◇
   ┌───▼──────┐   ┌────▼──────┐
   │ Ветка    │   │  Ветка    │
   │  «да»   │   │   «нет»  │
   └───┬──────┘   └────┬──────┘
       └────────┬───────┘
                │
           следующий блок
```

**Цикл («Пока»):**

```
       ◇──────────────◇
  нет◇  Условие цикла  ◇да
      ◇                ◇
      │         ┌───────▼──────┐
      │         │  Тело цикла  │
      │         └───────┬──────┘
      │                 │
      │         (назад к условию)◄──┘
      │
   следующий блок
```

---

## 4. ASCII-схемы как рабочий формат

Рисовать схемы в графических редакторах удобно, но не всегда практично. В учебных целях и при работе с текстом отлично работают **ASCII-схемы** — схемы из обычных символов.

Правила нашего ASCII-формата:
- Прямоугольники — из `─`, `│`, `┌`, `┐`, `└`, `┘`
- Овалы — из `╭`, `╮`, `╯`, `╰`
- Ромбы — приближённо из `◇` или угловых символов
- Стрелки — `→`, `↓`, `↑`, `←` или `▼`, `►`
- Схема помещается в блок кода (тройные обратные кавычки)

Это главный формат для схем в нашем учебнике. Mermaid-диаграммы — только если ASCII не справляется с задачей.

## 5. Канцтовары: блок-схема чека

Возьмём алгоритм из § 3.4 — печать строки чека с проверкой скидки. Запишем его блок-схемой:

*Алгоритм: напечатать строку чека. Если цена больше 500 рублей — применить скидку 10%.*

```
    ╭─────────────────────────────╮
    │           НАЧАЛО            │
    ╰──────────────┬──────────────╯
                   │
    ┌──────────────▼──────────────┐
    │  Получить: Товар, Цена      │
    └──────────────┬──────────────┘
                   │
          ◇────────▼────────◇
     да ◇    Цена > 500?     ◇ нет
        ◇                   ◇
   ┌────▼────────┐     ┌─────▼───────────┐
   │ Скидка =    │     │ Скидка = 0      │
   │ Цена × 10%  │     │                 │
   └────┬────────┘     └─────┬───────────┘
        └──────────┬──────────┘
                   │
    ┌──────────────▼──────────────┐
    │ ИтогоКОплате = Цена - Скидка│
    └──────────────┬──────────────┘
                   │
    ┌──────────────▼──────────────┐
    │ Сообщить: Товар, ИтогоКОплате│
    └──────────────┬──────────────┘
                   │
    ╭──────────────▼──────────────╮
    │            КОНЕЦ            │
    ╰─────────────────────────────╯
```

Теперь переведём схему в код:

```
Процедура НапечататьСтрокуЧека(Товар, Цена)
    Если Цена > 500 Тогда
        Скидка = Цена * 0.1;
    Иначе
        Скидка = 0;
    КонецЕсли;
    ИтогоКОплате = Цена - Скидка;
    Сообщить(Товар + ": " + ИтогоКОплате + " руб.");
КонецПроцедуры;
```

Схема и код — одно и то же. Схема читается быстрее; код — точнее и выполняется.

---

## 6. Ограничения блок-схем

Блок-схемы хорошо показывают *управляющий поток* — порядок выполнения и ветвления. Но они плохо показывают *данные*: что именно передаётся, как переменные связаны между собой. Для больших программ схема быстро становится громоздкой.

Поэтому блок-схему используют для фрагментов и ключевых алгоритмов, а не для всей программы целиком. Это инструмент для мышления и объяснения, а не документирования каждой строки.

---

## Главное в одном абзаце

Блок-схема — графическое представление алгоритма: овал для начала и конца, прямоугольник для действия, ромб для условия, стрелки для потока. Три конструкции структурного программирования (последовательность, ветвление, цикл) имеют стандартные изображения в блок-схемах — зная их, можно читать и рисовать схемы любой сложности. ASCII-схемы из обычных символов — удобный рабочий формат. Блок-схема полезна до написания кода (видна вся структура) и для объяснения (читается без знания языка). Но она не заменяет код и плохо подходит для описания данных.

## Упражнения

**Упражнение 1.** Прочитайте блок-схему и запишите соответствующий код на 1С:

```
    ╭──────────╮
    │  НАЧАЛО  │
    ╰─────┬────╯
          │
    ┌─────▼──────────────┐
    │  Счётчик = 1       │
    └─────┬──────────────┘
          │
        ◇─▼──────────────◇
  да  ◇   Счётчик <= 5?   ◇ нет
      ◇                   ◇
  ┌───▼────────────┐       │
  │  Сообщить      │       │
  │  (Счётчик)     │       │
  └───┬────────────┘   ╭───▼────╮
      │                │  КОНЕЦ │
  ┌───▼────────────┐   ╰────────╯
  │  Счётчик =     │
  │  Счётчик + 1   │
  └───┬────────────┘
      │
  (назад к условию)
```

---

**Упражнение 2.** Нарисуйте блок-схему для следующего кода:

```
Если Температура > 30 Тогда
    Сообщить("Жарко");
ИначеЕсли Температура > 15 Тогда
    Сообщить("Тепло");
Иначе
    Сообщить("Прохладно");
КонецЕсли;
```

---

**Упражнение 3.** Нарисуйте блок-схему для алгоритма: «Перебрать товары в списке. Для каждого: если цена больше 100 — сообщить название товара».

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

```
Счётчик = 1;
Пока Счётчик <= 5 Цикл
    Сообщить(Счётчик);
    Счётчик = Счётчик + 1;
КонецЦикла;
```

Схема полностью соответствует циклу «Пока». Инициализация перед циклом — в прямоугольнике. Условие — в ромбе. Тело и приближение к условию выхода — в двух прямоугольниках. Стрелка возврата — обратная дуга к ромбу.

---

**К упражнению 2.** *Указание.*

Два ромба, три ветки результата. Первый ромб: `Температура > 30?`. Ветка «да» — прямоугольник «Жарко» → конец. Ветка «нет» ведёт ко второму ромбу: `Температура > 15?`. Ветка «да» — «Тепло». Ветка «нет» — «Прохладно». Обе последние ветки сходятся к концу.

---

**К упражнению 3.** *Указание.*

Цикл «Для каждого Из» в блок-схеме выглядит как «Пока», где условие — «есть ещё элементы в списке». Внутри тела — ромб с проверкой `Цена > 100`. Ветка «да» — прямоугольник с `Сообщить`. Ветка «нет» — возврат к следующей итерации.

---

*Блок-схема показывает алгоритм статично: вот структура, вот ветки. Но она не показывает, как именно изменяются переменные в процессе работы. Для этого нужен другой инструмент — прогонка алгоритма на бумаге. О ней — § 3.7, последний параграф третьего модуля.*
