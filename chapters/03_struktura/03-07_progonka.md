## § 3.7. Прогонка алгоритмов на бумаге

В § 3.6 мы научились рисовать алгоритм — видеть его структуру целиком, до кода. Это взгляд сверху. Теперь — взгляд изнутри: как алгоритм работает шаг за шагом.

Компьютер выполняет код мгновенно и молча. Он не объясняет, что происходит с переменными на каждом шаге. **Прогонка** — способ сделать это самостоятельно: пройти по коду вручную, записывать состояние переменных после каждой команды и убедиться, что алгоритм делает именно то, что задумано.

> **Прогонка — это имитация работы компьютера на бумаге. Шаг за шагом, команда за командой.**

Этот навык не вспомогательный. Разработчики, которые умеют прогонять алгоритмы, находят ошибки до запуска — на этапе чтения кода.

---

## 1. Что такое прогонка

Прогонка — пошаговое выполнение кода вручную. На каждом шаге:
1. Берём следующую команду.
2. Выполняем её — мысленно или письменно.
3. Записываем, как изменились переменные.
4. Переходим к следующей команде.

«Следующая команда» определяется по правилам языка: при `Если` переходим в нужную ветку, при цикле — повторяем тело, пока условие истинно, при вызове подпрограммы — уходим внутрь неё.

---

## 2. Таблица состояния

Прогонку удобно вести в виде таблицы. Столбцы — переменные, которые нас интересуют, плюс столбец «Шаг» и «Экран» (для вывода `Сообщить`). Строка — одно изменение состояния.

Возьмём простой пример:

```
Сумма = 0;
Счётчик = 1;
Пока Счётчик <= 3 Цикл
    Сумма = Сумма + Счётчик;
    Счётчик = Счётчик + 1;
КонецЦикла;
Сообщить(Сумма);
```

Прогонка:

| Шаг | Команда                    | Сумма | Счётчик | Экран |
|-----|----------------------------|-------|---------|-------|
| 1   | `Сумма = 0`                | 0     | —       |       |
| 2   | `Счётчик = 1`              | 0     | 1       |       |
| 3   | Проверка: `1 <= 3` → Истина| 0     | 1       |       |
| 4   | `Сумма = 0 + 1`            | 1     | 1       |       |
| 5   | `Счётчик = 1 + 1`          | 1     | 2       |       |
| 6   | Проверка: `2 <= 3` → Истина| 1     | 2       |       |
| 7   | `Сумма = 1 + 2`            | 3     | 2       |       |
| 8   | `Счётчик = 2 + 1`          | 3     | 3       |       |
| 9   | Проверка: `3 <= 3` → Истина| 3     | 3       |       |
| 10  | `Сумма = 3 + 3`            | 6     | 3       |       |
| 11  | `Счётчик = 3 + 1`          | 6     | 4       |       |
| 12  | Проверка: `4 <= 3` → Ложь  | 6     | 4       |       |
| 13  | `Сообщить(6)`              | 6     | 4       | 6     |

Цикл прошёл три раза. Накопил сумму 1 + 2 + 3 = 6. На шаге 12 условие стало Ложью — цикл завершился. Результат на экране: `6`.

> **Таблица состояния — инструмент прогонки: строка за строкой фиксируем значения переменных после каждого изменения.**

---

## 3. Прогонка с ветвлением

Когда в коде есть `Если`, прогонка требует выбора: в какую ветку мы идём? Это определяет условие на текущем шаге.

```
Цена = 400;
Если Цена > 500 Тогда
    Скидка = Цена * 0.1;
Иначе
    Скидка = 0;
КонецЕсли;
ИтогоКОплате = Цена - Скидка;
Сообщить(ИтогоКОплате);
```

| Шаг | Команда                         | Цена | Скидка | ИтогоКОплате | Экран |
|-----|---------------------------------|------|--------|--------------|-------|
| 1   | `Цена = 400`                    | 400  | —      | —            |       |
| 2   | Проверка: `400 > 500` → Ложь    | 400  | —      | —            |       |
| 3   | `Скидка = 0` (ветка «Иначе»)   | 400  | 0      | —            |       |
| 4   | `ИтогоКОплате = 400 - 0`        | 400  | 0      | 400          |       |
| 5   | `Сообщить(400)`                 | 400  | 0      | 400          | 400   |

Условие `Цена > 500` не выполнилось — перешли в `Иначе`. Скидки нет, цена прошла без изменений.

---

## 4. Прогонка с вызовом подпрограммы

При вызове подпрограммы прогонка «уходит внутрь» неё — мы продолжаем таблицу, но теперь следим за переменными внутри подпрограммы. После выхода из неё возвращаемся к основному потоку.

```
Функция МаксимумДвух(А, Б)
    Если А >= Б Тогда
        Возврат А;
    Иначе
        Возврат Б;
    КонецЕсли;
КонецФункции;

Результат = МаксимумДвух(7, 12);
Сообщить(Результат);
```

| Шаг | Где                   | Команда                       | А  | Б  | Результат | Возвр. | Экран |
|-----|-----------------------|-------------------------------|----|----|-----------|--------|-------|
| 1   | основной код          | вызов `МаксимумДвух(7, 12)`   | —  | —  | —         |        |       |
| 2   | внутри функции        | `А = 7`, `Б = 12` (параметры) | 7  | 12 | —         |        |       |
| 3   | внутри функции        | Проверка: `7 >= 12` → Ложь    | 7  | 12 | —         |        |       |
| 4   | внутри функции        | `Возврат 12`                  | 7  | 12 | —         | 12     |       |
| 5   | основной код          | `Результат = 12`              | —  | —  | 12        |        |       |
| 6   | основной код          | `Сообщить(12)`                | —  | —  | 12        |        | 12    |

На шаге 1 выполнение уходит внутрь функции. На шаге 4 функция возвращает `12`. На шаге 5 это значение присваивается `Результат`.

---

## 5. Как прогонка помогает находить ошибки

Разберём типичную ошибку. Программист написал функцию «максимум» неправильно:

```
Функция МаксимумДвух(А, Б)
    Если А > Б Тогда
        Возврат Б;  // ошибка: возвращаем не то
    Иначе
        Возврат А;
    КонецЕсли;
КонецФункции;
```

Прогоняем `МаксимумДвух(7, 12)`:

| Шаг | Команда                    | А | Б  | Возвр. |
|-----|----------------------------|---|----|--------|
| 1   | параметры: А=7, Б=12       | 7 | 12 |        |
| 2   | Проверка: `7 > 12` → Ложь  | 7 | 12 |        |
| 3   | `Возврат А` → `Возврат 7`  | 7 | 12 | 7      |

Результат: 7. Но максимум из 7 и 12 — это 12. Прогонка показала ошибку до запуска: в ветке `Если` возвращается `Б` (меньшее), а в `Иначе` — `А`. Всё наоборот.

> **Прогонка обнаруживает ошибки раньше компьютера. Это быстрее, чем запускать программу и догадываться, что пошло не так.**

---

## 6. Когда прогонять

Прогонка — не обязательный ритуал для каждой строки. Её применяют:

- **При непонимании.** Если не ясно, что делает незнакомый код — прогоняем.
- **При отладке.** Программа ведёт себя не так, как ожидалось — прогоняем, чтобы найти, где расходятся ожидание и реальность.
- **При написании.** Написали новый алгоритм — быстрая прогонка проверяет, что он делает то, что задумано, хотя бы на одном примере.

Опытные разработчики прогоняют простые фрагменты мысленно — не записывая таблицу. Запись нужна, когда код сложный или нужно объяснить другому.

## 7. Канцтовары: прогонка цикла по списку товаров

Прогоняем программу из § 3.3 — подсчёт суммы заказа:

```
Товары = Новый Массив;
Товары.Добавить(35);
Товары.Добавить(60);
Товары.Добавить(15);

Итого = 0;
Для каждого Цена Из Товары Цикл
    Итого = Итого + Цена;
КонецЦикла;
Сообщить(Итого);
```

Массив: `[35, 60, 15]`. Прогонка:

| Шаг | Команда                         | Цена (текущая) | Итого | Экран |
|-----|---------------------------------|----------------|-------|-------|
| 1–4 | инициализация массива и Итого   | —              | 0     |       |
| 5   | итерация 1: `Цена = 35`         | 35             | 0     |       |
| 6   | `Итого = 0 + 35`                | 35             | 35    |       |
| 7   | итерация 2: `Цена = 60`         | 60             | 35    |       |
| 8   | `Итого = 35 + 60`               | 60             | 95    |       |
| 9   | итерация 3: `Цена = 15`         | 15             | 95    |       |
| 10  | `Итого = 95 + 15`               | 15             | 110   |       |
| 11  | элементов больше нет → выход    | —              | 110   |       |
| 12  | `Сообщить(110)`                 | —              | 110   | 110   |

Три итерации, три добавления. Итог: 110 рублей. Прогонка подтверждает: алгоритм накапливает сумму правильно.

---

## Главное в одном абзаце

Прогонка — пошаговое ручное выполнение алгоритма: берём очередную команду, выполняем её согласно правилам языка, фиксируем изменения переменных в таблице состояния. При `Если` — идём в нужную ветку; при цикле — повторяем тело, пока условие истинно; при вызове подпрограммы — уходим внутрь неё и возвращаемся с результатом. Прогонка позволяет найти ошибки до запуска: несоответствие между тем, что написано, и тем, что задумывалось, становится видно в таблице. Опытные разработчики прогоняют простые фрагменты мысленно; запись нужна для сложных случаев и для объяснения.

**К упражнению 1.** *Полное решение.*

|-----|-------------------------------|---|--------------|-------|
| 1   | `Н = 1`                       | 1 | —            |       |

**К упражнению 2.** *Полное решение.*

Прогонка входа в `СуммаДвух(10, 20)`: параметры `А = 10`, `Б = 20`. Выполняется `Результат = 10 + 20 = 30`. Затем — конец функции.

Ошибка: нет команды `Возврат`. Функция ничего не возвращает. Переменная `Итог` получит не значение, а ничего (в 1С — `Неопределено`). `Сообщить` выведет `Неопределено`.

Исправление — добавить `Возврат`:

```
Функция СуммаДвух(А, Б)
    Результат = А + Б;
    Возврат Результат;
КонецФункции;
```

## Итог Модуля 3

Мы прошли через весь третий модуль — и это был большой путь.

В начале (§ 3.1) мы увидели три кита структурного программирования: последовательность, ветвление и цикл. Это не просто конструкции языка — это универсальные паттерны, из которых строится любой алгоритм. В §§ 3.2–3.3 мы разобрали каждый вид цикла детально: «Пока» — управляемый условием, «Для каждого Из» — управляемый коллекцией.

В §§ 3.4–3.5 мы добавили к этому подпрограммы: процедуры и функции. Теперь у нас есть не только конструкции — у нас есть способ дать им имя, вызывать повторно и изолировать в отдельные блоки. Вместе с параметрами это даёт гибкость: один код — разные данные.

В §§ 3.6–3.7 мы освоили два инструмента мышления: блок-схему для проектирования и объяснения, прогонку для проверки и отладки.

Четвёртый модуль — паттерны: типовые фразеологизмы, которые встречаются в реальном коде снова и снова. Умея читать отдельные конструкции, вы научитесь узнавать готовые смысловые блоки.
