## § 3.5. Параметры подробно: передача и область видимости

В § 3.4 мы познакомились с параметрами: переменными, объявленными в заголовке подпрограммы и получающими значения при вызове. Там же была оговорка: параметры — тема с тонкостями. Сейчас разберёмся с этими тонкостями.

Два вопроса, которые поставит этот параграф. Первый: что именно передаётся в параметр — само значение или что-то другое? Второй: если внутри подпрограммы создать переменную, будет ли она видна снаружи?

---

## 1. Два способа передачи

Представьте, что у вас есть блокнот с записанным числом `100`. Вы просите коллегу что-то посчитать.

**Способ А:** вы даёте ему *ксерокопию листа*. Он пишет на копии. Ваш блокнот остаётся нетронутым.

**Способ Б:** вы даёте ему *сам блокнот*. Он пишет прямо в нём. Когда он вернёт блокнот — в нём окажется то, что он написал.

Именно так работают два способа передачи параметров:

- **Передача по значению** — подпрограмма получает копию. Изменения внутри не затрагивают оригинал.
- **Передача по ссылке** — подпрограмма работает с оригинальной переменной. Изменения внутри отражаются в вызывающем коде.

---

## 2. По умолчанию — передача по ссылке

Большинство популярных языков программирования по умолчанию передают числа и строки по значению. В 1С **иначе**: по умолчанию передача происходит по ссылке.

Посмотрим на пример:

```
Процедура УвеличитьНаДесять(Число)
    Число = Число + 10;
КонецПроцедуры;

Х = 5;
УвеличитьНаДесять(Х);
Сообщить(Х);
```

Что выведет последняя строка? Можно предположить `5` — ведь мы передали значение в процедуру, а не сам `Х`. Но выведет `15`.

Причина: параметр `Число` в этом примере — не копия переменной `Х`, а другое имя для той же переменной. Когда процедура пишет `Число = Число + 10`, она изменяет именно `Х`.

> **В 1С параметр по умолчанию — это псевдоним (другое имя) для переменной, переданной при вызове. Изменение параметра внутри подпрограммы изменяет оригинальную переменную.**

Это важно запомнить: поведение 1С здесь неочевидно для тех, кто знаком с другими языками, и непредсказуемо для тех, кто не знает правила.

---

## 3. Ключевое слово `Знач`: передача по значению

Чтобы подпрограмма получила копию и не могла изменить оригинал, перед именем параметра в заголовке ставится ключевое слово **`Знач`**:

```
Процедура УвеличитьНаДесять(Знач Число)
    Число = Число + 10;
КонецПроцедуры;

Х = 5;
УвеличитьНаДесять(Х);
Сообщить(Х);
```

Теперь последняя строка выведет `5`. Процедура получила копию значения `5`, увеличила копию до `15`, но оригинальная переменная `Х` осталась нетронутой.

```
  Без Знач (по ссылке)          С Знач (по значению)
  ─────────────────────────     ─────────────────────────
  Х = 5                         Х = 5
  вызов → Число = Х (тот же)   вызов → Число = копия 5
  Число = 15                    Число = 15
  выход → Х = 15 (изменился)   выход → Х = 5 (не изменился)
```

> **`Знач` перед именем параметра означает «передать копию». Подпрограмма может изменять параметр как угодно — оригинал остаётся нетронутым.**

Когда использовать `Знач`? Когда подпрограмме нужно работать с переданным значением как с рабочим материалом — менять, накапливать — и при этом не затрагивать переменную вызывающего кода. Если изменение параметра не предполагается вообще, `Знач` — хороший способ выразить это намерение явно.

---

## 4. Область видимости: локальные переменные

Любая переменная, созданная внутри подпрограммы, **локальна**: она существует только в пределах этой подпрограммы. Как только подпрограмма завершается — переменная исчезает.

```
Процедура ПосчитатьЧто-нибудь()
    ВременныйРезультат = 42;
    Сообщить(ВременныйРезультат);
КонецПроцедуры;

ПосчитатьЧто-нибудь();
Сообщить(ВременныйРезультат); // ошибка: переменная не определена
```

Переменная `ВременныйРезультат` создана внутри процедуры — снаружи она недоступна. Попытка обратиться к ней после вызова вызовет ошибку.

> **Переменные, объявленные внутри подпрограммы, локальны. Они существуют только на время работы подпрограммы и недоступны снаружи.**

Это защита: подпрограмма не засоряет пространство имён вокруг. Сколько бы переменных она ни создавала внутри — по завершении от них не остаётся следа.

---

## 5. Область видимости: что видно изнутри

Снаружи — в общем коде модуля — могут быть объявлены переменные, не принадлежащие ни одной подпрограмме. Такие переменные называют **переменными модуля** (или глобальными в контексте модуля). Они видны из любой подпрограммы этого же модуля.

```
КурсВалюты = 92;

Функция ПересчитатьВРубли(СуммаВДолларах)
    Возврат СуммаВДолларах * КурсВалюты; // КурсВалюты видна здесь
КонецФункции;

Сообщить(ПересчитатьВРубли(10)); // 920
```

Переменная `КурсВалюты` объявлена вне любой подпрограммы. Функция `ПересчитатьВРубли` видит её и использует.

Однако здесь есть риск: если внутри подпрограммы создать переменную с тем же именем, что у переменной модуля, внутренняя **перекроет** внешнюю — и обратиться к внешней из подпрограммы уже не получится. Это называют **затенением**. Способ избежать путаницы — давать локальным переменным и переменным модуля непересекающиеся имена.

```
  Что видно изнутри подпрограммы:
  ─────────────────────────────────────────
  ✓ параметры подпрограммы
  ✓ локальные переменные (созданные внутри)
  ✓ переменные модуля (объявленные снаружи)
    — если не перекрыты локальной с тем же именем
```

---

## 6. Параметры: обязательные и необязательные

В § 3.4 упоминалось, что у встроенных подпрограмм бывают необязательные параметры. В пользовательских подпрограммах это тоже возможно — через **значения по умолчанию**.

Если при объявлении параметра указать значение через `=`, он становится необязательным: при вызове его можно опустить, и тогда он автоматически получит указанное значение.

```
Функция Сложить(А, Б = 0)
    Возврат А + Б;
КонецФункции;

Сложить(5);       // OK: Б = 0 (значение по умолчанию)
Сложить(5, 3);    // OK: Б = 3
```

Параметры без значения по умолчанию — **обязательные**: передать их при вызове необходимо. Параметры со значением по умолчанию — **необязательные**: можно опустить.

Важное ограничение: необязательные параметры должны идти *после* обязательных. Нельзя поставить необязательный параметр раньше обязательного — платформа не сможет понять, какой именно аргумент был пропущен.

```
// Корректно: сначала обязательные, потом необязательные
Функция НапечататьЧек(Товар, Цена, ПроцентСкидки = 0)
    ...
КонецФункции;

НапечататьЧек("Ручка", 35);       // ПроцентСкидки = 0
НапечататьЧек("Тетрадь", 60, 10); // ПроцентСкидки = 10
```

Передать меньше аргументов, чем обязательных параметров, или больше, чем всего параметров — ошибка.

## 7. Канцтовары: передача и область видимости в действии

Рассмотрим программу, которая вычисляет итоговую сумму заказа. Покажем, как работают по ссылке и по значению вместе:

```
КурсДоставки = 150;

Функция ВычислитьИтого(Знач СуммаТоваров)
    СуммаТоваров = СуммаТоваров + КурсДоставки; // прибавляем доставку
    Возврат СуммаТоваров;
КонецФункции;

СуммаЗаказа = 800;
Итого = ВычислитьИтого(СуммаЗаказа);

Сообщить("Товары: " + СуммаЗаказа + " руб."); // 800 — не изменилась
Сообщить("Итого с доставкой: " + Итого + " руб."); // 950
```

Разберём:

- `КурсДоставки` — переменная модуля. Функция `ВычислитьИтого` видит её напрямую.
- `СуммаТоваров` объявлен с `Знач` — функция получает копию `800`. Работает с копией, увеличивает её до `950`, возвращает.
- Оригинальная переменная `СуммаЗаказа` остаётся `800`.
- Итог: 950 = 800 + 150.

На экране:
```
Товары: 800 руб.
Итого с доставкой: 950 руб.
```

---

## Главное в одном абзаце

В 1С параметры по умолчанию передаются по ссылке: параметр внутри подпрограммы — это другое имя для оригинальной переменной, и изменение параметра изменяет оригинал. Ключевое слово `Знач` перед именем параметра переключает в режим передачи по значению: подпрограмма получает копию, оригинал остаётся нетронутым. Переменные, созданные внутри подпрограммы, локальны — они исчезают при завершении подпрограммы и недоступны снаружи. Переменные модуля, объявленные вне подпрограмм, видны изнутри любой подпрограммы этого же модуля, если не перекрыты локальной переменной с тем же именем. Количество аргументов при вызове должно быть не меньше числа обязательных параметров и не больше общего числа параметров.

## Упражнения

**Упражнение 1.** Определите, что выведет программа:

```
Процедура Удвоить(Число)
    Число = Число * 2;
КонецПроцедуры;

Х = 7;
Удвоить(Х);
Сообщить(Х);
```

---

**Упражнение 2.** Та же процедура, но с `Знач`. Что изменится?

```
Процедура Удвоить(Знач Число)
    Число = Число * 2;
КонецПроцедуры;

Х = 7;
Удвоить(Х);
Сообщить(Х);
```

---

**Упражнение 3.** Найдите потенциальную проблему в коде:

```
Скидка = 10;

Процедура НапечататьЦену(Цена)
    Скидка = Цена * 0.1;
    Сообщить("Цена со скидкой: " + (Цена - Скидка) + " руб.");
КонецПроцедуры;

НапечататьЦену(500);
Сообщить("Текущая скидка: " + Скидка + " руб.");
```

Какое значение выведет последняя строка? Почему это может быть неожиданным?

---

**Упражнение 4.** В «Канцтоварах» нужна функция, которая добавляет налог к цене. Напишите `ДобавитьНалог(Цена, ПроцентНалога)`, используя `Знач` там, где это нужно. Проверьте: `ДобавитьНалог(1000, 20)` должно вернуть `1200`, а оригинальная переменная цены — не измениться.

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Параметр `Число` объявлен без `Знач` — передача по ссылке. Внутри процедуры `Число = Число * 2` изменяет не копию, а оригинальную переменную `Х`. После вызова `Х = 14`.

Выведет: **14**.

---

**К упражнению 2.** *Полное решение.*

Теперь `Знач` — передача по значению. Процедура получает копию `7`. Внутри `Число = 14`, но это изменение касается только копии. Оригинальный `Х` остаётся `7`.

Выведет: **7**.

---

**К упражнению 3.** *Полное решение.*

Внутри `НапечататьЦену` создаётся локальная переменная `Скидка`. Она перекрывает переменную модуля с тем же именем. Внутри процедуры `Скидка = 500 * 0.1 = 50` — это локальная переменная.

Но! Поскольку `Скидка` создана без `Знач` и является локальной, она не связана с внешней `Скидка = 10`. После завершения процедуры локальная `Скидка` исчезает.

Последняя строка выведет: **10** — переменная модуля не изменилась.

Проблема не в результате, а в читаемости: одноимённые переменные внутри и снаружи — источник путаницы. Лучше назвать локальную переменную иначе, например `СуммаСкидки`.

---

**К упражнению 4.** *Полное решение.*

```
Функция ДобавитьНалог(Знач Цена, Знач ПроцентНалога)
    Цена = Цена + Цена * ПроцентНалога / 100;
    Возврат Цена;
КонецФункции;

МояЦена = 1000;
ЦенаСНалогом = ДобавитьНалог(МояЦена, 20);
Сообщить("Исходная цена: " + МояЦена + " руб."); // 1000
Сообщить("С налогом: " + ЦенаСНалогом + " руб."); // 1200
```

`Знач` перед обоими параметрами: функция работает с копиями. `МояЦена` остаётся `1000`. Внутри копия `Цена` изменяется до `1200` и возвращается через `Возврат`.
