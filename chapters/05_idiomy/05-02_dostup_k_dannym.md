## § 5.2. Обращение к данным: документы и справочники

В § 5.1 мы разобрали, что такое объект: значение с внутренней структурой, к которой обращаются через точку. Массив — самый простой пример такого объекта. Но в реальной конфигурации 1С живут совсем другие объекты: справочники, документы, регистры — именно те, о которых мы говорили в § 0.5. Они уже существуют в базе данных. Как до них добраться?

---

## 1. Менеджер объекта

К объектам конфигурации обращаются не напрямую по имени переменной — их нужно сначала найти или получить. Для этого у каждого вида объектов есть **менеджер** — особый объект, который знает, как работать со всеми записями этого вида.

Менеджеры в 1С называются через точку по роду объекта:

```
Справочники.Номенклатура         // менеджер справочника «Номенклатура»
Документы.РеализацияТоваров      // менеджер документа «РеализацияТоваров»
```

Менеджер не является конкретной записью — это, скорее, «стойка выдачи». Через неё можно найти нужную запись или перебрать все.

> **Менеджер — объект, через который ищут, создают и перебирают записи определённого вида. Справочники.ИмяСправочника, Документы.ИмяДокумента.**

---

## 2. Поиск записи

У менеджера есть методы поиска. Самые частые — поиск по наименованию и по коду:

```
// Найти элемент справочника по наименованию
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию("Ручка");

// Найти элемент справочника по коду
Ссылка = Справочники.Номенклатура.НайтиПоКоду("000001");

// Найти документ по номеру
Ссылка = Документы.РеализацияТоваров.НайтиПоНомеру("РТ-000042");
```

Эти методы возвращают **ссылку** на найденный объект. Ссылка — не сам объект, а указатель на него в базе данных. Что такое ссылка и зачем она нужна — подробно в § 5.4; пока воспринимайте её как «результат поиска, с которым можно работать дальше».

Если объект не найден, метод возвращает *пустую ссылку* — специальное значение, означающее «ничего не нашли». Это аналог `Неопределено` из паттерна поиска (§ 4.2), только специфичный для объектов 1С.

---

## 3. Проверка результата поиска

После поиска — обязательная проверка. Инструмент — встроенная функция `ЗначениеЗаполнено()`: она возвращает `Истина`, если значение не пустое, и `Ложь`, если пустое.

```
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию("Ручка");

Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить("Товар не найден");
    Возврат;
КонецЕсли;

// здесь знаем: Ссылка не пустая
Сообщить("Найден: " + Ссылка.Наименование);
```

`ЗначениеЗаполнено()` — одна из самых частых функций в реальном коде 1С. Она работает не только со ссылками, но и с числами (0 — не заполнено), строками (пустая строка — не заполнено), датами (пустая дата — не заполнено). Подробнее она появится в § 5.5.

> **После любого поиска — проверить результат через `ЗначениеЗаполнено()`. Это защитный код (§ 4.5), обязательная часть паттерна.**

---

## 4. Чтение реквизитов

Когда ссылка получена, её реквизиты (свойства) читаются напрямую через точку — совсем как в § 5.1:

```
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию("Ручка");

Если ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить(Ссылка.Наименование);
    Сообщить(Ссылка.Артикул);
    Сообщить(Ссылка.ЕдиницаИзмерения.Наименование); // цепочка точек
КонецЕсли;
```

Реквизиты справочника `Номенклатура` — это те поля, которые вы видели бы в карточке товара: наименование, артикул, единица измерения. Единица измерения сама является ссылкой на другой справочник, поэтому дальше снова идёт точка.

---

## 5. Перебор всех записей

Менеджер позволяет не только найти конкретную запись, но и перебрать все. Для этого используют метод `.Выбрать()`, который возвращает специальный объект-выборку, и цикл:

```
Выборка = Справочники.Номенклатура.Выбрать();
Пока Выборка.Следующий() Цикл
    Сообщить(Выборка.Наименование);
КонецЦикла;
```

Здесь `Пока` — цикл из § 3.2. Метод `.Следующий()` переходит к следующей записи и возвращает `Истина`, пока записи не закончились. Когда записи закончились — возвращает `Ложь`, и цикл завершается.

Это стандартный оборот для обхода всех записей справочника или документов. В § 5.5 он войдёт в каталог типичных идиом.

---

## 6. Канцтовары: найти товар и прочитать его данные

Программа ищет товар «Тетрадь» в справочнике Номенклатура и выводит его данные:

```
НазваниеТовара = "Тетрадь";
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию(НазваниеТовара);

Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить("Товар «" + НазваниеТовара + "» не найден в справочнике");
    Возврат;
КонецЕсли;

Сообщить("Товар: " + Ссылка.Наименование);
Сообщить("Артикул: " + Ссылка.Артикул);
```

Разберём по шагам:

1. `НайтиПоНаименованию("Тетрадь")` — запрос к базе данных, возвращает ссылку или пустую ссылку.
2. `НЕ ЗначениеЗаполнено(Ссылка)` — сторожевая проверка: если не нашли, сообщить и выйти.
3. Если нашли — читать реквизиты через точку.

Форма та же, что в паттерне поиска из § 4.2, только вместо цикла — метод менеджера. Сторожевая проверка та же, что в § 4.5.

---

## Главное в одном абзаце

Объекты конфигурации (справочники, документы) доступны через менеджеры: `Справочники.ИмяСправочника`, `Документы.ИмяДокумента`. Менеджер предоставляет методы поиска — `НайтиПоНаименованию()`, `НайтиПоКоду()`, `НайтиПоНомеру()` — и метод `Выбрать()` для перебора всех записей. Методы поиска возвращают ссылку или пустую ссылку. После поиска — обязательная проверка через `ЗначениеЗаполнено()`: это та же сторожевая проверка из защитного кода, только применённая к объектам платформы. Реквизиты найденного объекта читаются через точку напрямую от ссылки; если реквизит сам является ссылкой — цепочка продолжается.

```
Выборка = Документы.РеализацияТоваров.Выбрать();
Количество = 0;
Пока Выборка.Следующий() Цикл
    Если Выборка.Проведен Тогда

КонецЦикла;
Сообщить("Проведённых документов: " + Количество);
```

Какой паттерн из Модуля 4 здесь используется?

**Упражнение 3.** Найдите потенциальную проблему в коде:

```
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию("Линейка");
Сообщить("Артикул: " + Ссылка.Артикул);
```

Что произойдёт, если «Линейки» нет в справочнике?

---

**Упражнение 4.** В «Канцтоварах» нужно вывести наименования всех товаров из справочника `Номенклатура`, у которых реквизит `ВидТовара` равен «Канцелярия». Напишите программу.

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

```
Ссылка = Справочники.Контрагенты.НайтиПоНаименованию("ООО Ромашка");

Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить("Контрагент «ООО Ромашка» не найден");
    Возврат;
КонецЕсли;

Сообщить("ИНН: " + Ссылка.ИНН);
```

---

**К упражнению 2.** *Полное решение.*

Программа перебирает все документы вида «РеализацияТоваров» и считает, сколько из них проведено (реквизит `Проведен = Истина`).

Паттерн: **агрегация** — счётчик. До цикла `Количество = 0`, внутри цикла условие и прибавление 1. Дополнительно — перебор через выборку вместо массива, но структура та же, что в § 4.4.

---

**К упражнению 3.** *Полное решение.*

Если «Линейки» нет, метод `НайтиПоНаименованию` вернёт пустую ссылку. Обращение к реквизиту `Артикул` пустой ссылки вызовет ошибку выполнения.

Исправление — сторожевая проверка:

```
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию("Линейка");
Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить("Товар не найден");
    Возврат;
КонецЕсли;
Сообщить("Артикул: " + Ссылка.Артикул);
```

---

**К упражнению 4.** *Указание.*

Используйте `Выбрать()` и цикл `Пока`. Внутри цикла проверьте `Выборка.ВидТовара = "Канцелярия"`. Если условие выполняется — выведите наименование. Это паттерн перебора с фильтрацией (§ 4.3).

---

*Мы научились находить записи и читать их реквизиты. Но у документов есть особенность, которой нет у простых переменных: внутри документа бывает не одно поле, а целая таблица строк — табличная часть. Это особый вид коллекции, и разбирается она в § 5.3.*
