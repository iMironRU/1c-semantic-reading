## § 5.1. Объекты, свойства и методы

В Модуле 4 мы работали с массивами, которые создавали сами: `Новый Массив`, `.Добавить()`, `.Количество()`. Оба метода появлялись мимоходом — пришло время объяснить, что за ними стоит. В реальном коде 1С большинство интересных значений устроены именно так: внутри них есть структура, и к этой структуре обращаются через точку. Понять точку — значит прочитать девяносто процентов того, что выглядит загадочно в чужом коде.

---

## 1. Значение с внутренней структурой

До сих пор все значения, с которыми мы работали, были «плоскими»: число `42`, строка `"Ручка"`, дата. Одно значение — одно содержимое, ничего внутри.

Но некоторые значения устроены иначе: у них есть *внутренняя структура* — именованные части, к которым можно обратиться по имени. Такие значения называют **объектами**.

Бытовой аналог — папка с документами. Папка — одна, но внутри неё несколько отделений с разными названиями: «Накладные», «Счета», «Акты». Чтобы добраться до конкретного содержимого, надо назвать нужное отделение.

В 1С обращение к внутренней части объекта записывается через точку:

```
Объект.ИмяЧасти
```

Точка здесь — операция «загляни внутрь».

---

## 2. Свойства: данные объекта

Именованная часть объекта, которая хранит какое-то значение, называется **свойством**. Свойство читается через точку точно так же, как обычная переменная — только перед точкой стоит имя объекта.

```
Сообщить(Документ.Номер);    // прочитать свойство Номер
Сообщить(Документ.Дата);     // прочитать свойство Дата
```

Некоторые свойства доступны только для чтения — платформа не разрешает их изменить напрямую. Другие можно и читать, и записывать:

```
Объект.Комментарий = "Срочно"; // записать в свойство
```

> **Свойство — именованная часть объекта, хранящая значение. Читается и (иногда) записывается через точку: `Объект.Свойство`.**

---

## 3. Методы: действия объекта

Кроме свойств, объект может уметь что-то *делать*. Именованное действие объекта называется **методом**. Внешне метод похож на обычную процедуру или функцию, но привязан к конкретному объекту и вызывается через точку.

```
Объект.Записать();           // метод без аргументов
Массив.Добавить("Ручка");    // метод с аргументом
```

Методы бывают двух видов — точно как подпрограммы из § 3.4:

- **Метод-процедура**: выполняет действие, ничего не возвращает.
- **Метод-функция**: выполняет действие *и* возвращает значение.

```
Массив.Добавить("Ручка");           // процедура: добавляет, не возвращает
Количество = Массив.Количество();   // функция: возвращает число
```

> **Метод — именованное действие объекта. Вызывается через точку: `Объект.Метод()`. Может принимать аргументы и возвращать значение.**

---

## 4. Точка как операция «загляни внутрь»

Точка не просто синтаксический разделитель — это операция. Она говорит: «возьми объект слева и найди в нём то, что названо справа».

```
Массив.Количество()
│      │
│      └── что ищем внутри: метод Количество
└── в каком объекте: Массив
```

Точки можно ставить в цепочку: если свойство объекта само является объектом — к нему тоже можно обратиться через точку.

```
Документ.Контрагент.Наименование
│         │          │
│         │          └── свойство объекта-контрагента
│         └── свойство документа (само является объектом)
└── исходный объект
```

Такие цепочки часто встречаются в реальном коде. Читать их нужно слева направо: «у документа возьми контрагента, у контрагента возьми наименование».

> **Цепочка точек читается слева направо: каждая точка означает «загляни внутрь» следующего объекта.**

---

## 5. Ключевое слово «Новый»

Мы уже использовали `Новый Массив` начиная с § 3.3 — мимоходом, без объяснения. Пришло время разобраться.

**`Новый`** — ключевое слово, которое создаёт новый объект указанного типа. После `Новый` пишется имя типа объекта, который нужно создать:

```
Товары = Новый Массив;
Список = Новый СписокЗначений;
Таблица = Новый ТаблицаЗначений;
```

Результат — новый, пустой объект этого типа. Дальше с ним работают через методы и свойства.

Не все объекты создаются через `Новый`. Объекты конфигурации — документы, элементы справочников — не создаются так; к ним обращаются через менеджеры (об этом в § 5.2). Но вспомогательные объекты-коллекции (`Массив`, `СписокЗначений`, `ТаблицаЗначений`) — именно через `Новый`.

---

## 6. Массив с новыми глазами

Теперь перечитаем код из Модуля 4 — уже понимая, что за ним стоит:

```
Цены = Новый Массив;          // создать новый объект типа «Массив»
Цены.Добавить(35);            // вызвать метод-процедуру Добавить
Цены.Добавить(60);
Цены.Добавить(15);

Сообщить(Цены.Количество());  // вызвать метод-функцию Количество
```

Прогонка:

| Шаг | Команда | Цены | Экран |
|-----|---------|------|-------|
| 1 | `Новый Массив` | пустой массив | |
| 2 | `.Добавить(35)` | [35] | |
| 3 | `.Добавить(60)` | [35, 60] | |
| 4 | `.Добавить(15)` | [35, 60, 15] | |
| 5 | `.Количество()` | [35, 60, 15] | `3` |

Метод `.Добавить()` — процедура: добавляет элемент, ничего не возвращает. Метод `.Количество()` — функция: возвращает число элементов. Именно это число и получает `Сообщить`.

У массива есть и другие методы. Два из них появятся в § 5.5. Пока главное — научиться видеть точку и понимать, что за ней.

---

## 7. Методы против функций-процедур

Важное различие для чтения кода: не всё, что похоже на вызов подпрограммы, является методом.

```
СтрДлина("Ручка")      // глобальная функция — НЕ метод
"Ручка".Длина()        // так в 1С НЕ пишут
Массив.Количество()    // метод объекта — пишут через точку
```

В 1С есть глобальные функции (`СтрДлина`, `Найти`, `Тип`, `ЗначениеЗаполнено`) и есть методы объектов. Глобальные функции вызываются по имени, без точки; методы — только через объект и точку. В коде это сразу видно: если перед скобками нет точки — это глобальная функция.

---

## 8. Канцтовары: объект в действии

Допустим, у нас есть объект «Заказ» с несколькими свойствами и методом:

```
// Читаем свойства заказа
Сообщить("Номер заказа: " + Заказ.Номер);
Сообщить("Дата: " + Заказ.Дата);
Сообщить("Клиент: " + Заказ.Контрагент.Наименование);

// Работаем с коллекцией товаров внутри заказа
Сообщить("Позиций в заказе: " + Заказ.Товары.Количество());

// Сохраняем изменения
Заказ.Комментарий = "Срочный";
Заказ.Записать();
```

В этом фрагменте:
- `Заказ.Номер`, `Заказ.Дата`, `Заказ.Комментарий` — свойства.
- `Заказ.Контрагент` — свойство, само являющееся объектом.
- `Заказ.Контрагент.Наименование` — цепочка: свойство объекта-контрагента.
- `Заказ.Товары.Количество()` — цепочка: метод коллекции-свойства.
- `Заказ.Записать()` — метод-процедура, сохраняет объект в базу.

---

## Главное в одном абзаце

Объект — это значение с внутренней структурой: в нём есть именованные части. Свойство — именованная часть, хранящая значение; читается и иногда записывается через точку. Метод — именованное действие; вызывается через точку со скобками. Методы бывают двух видов: процедуры (только действуют) и функции (действуют и возвращают значение). Точка — операция «загляни внутрь»; точки можно ставить в цепочку. `Новый ИмяТипа` создаёт новый пустой объект — так создаются вспомогательные коллекции. Глобальные функции (`СтрДлина`, `Найти`) не являются методами — они вызываются без объекта и точки.

## Упражнения

**Упражнение 1.** В следующем коде выделите каждую строку: что здесь — свойство, что — вызов метода, что — глобальная функция?

```
Список = Новый СписокЗначений;
Список.Добавить("Ручка");
Список.Добавить("Карандаш");
Н = Список.Количество();
Сообщить("Позиций: " + Н);
Д = СтрДлина("Карандаш");
Сообщить(Д);
```

---

**Упражнение 2.** Прочитайте цепочку и объясните её на русском языке шаг за шагом:

```
Сообщить(Документ.Организация.ИНН);
```

Что такое `Документ`, `Организация`, `ИНН` в этой записи?

---

**Упражнение 3.** Найдите ошибку в рассуждении: «`.Количество()` — это функция, которая считает количество элементов массива. Значит, `СтрДлина()` — это метод строки».

---

**Упражнение 4.** Напишите программу: создать массив, добавить три товара («Ручка», «Тетрадь», «Ластик»), вывести количество элементов и второй элемент. Для получения второго элемента используйте метод `.Получить(Индекс)` — он возвращает элемент по порядковому номеру (нумерация с нуля).

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

```
Список = Новый СписокЗначений;  // создание объекта
Список.Добавить("Ручка");       // вызов метода-процедуры
Список.Добавить("Карандаш");    // вызов метода-процедуры
Н = Список.Количество();        // вызов метода-функции; результат в переменную
Сообщить("Позиций: " + Н);      // глобальная функция Сообщить
Д = СтрДлина("Карандаш");       // глобальная функция СтрДлина
Сообщить(Д);                    // глобальная функция Сообщить
```

`СтрДлина` — глобальная функция, а не метод: перед ней нет объекта и точки.

---

**К упражнению 2.** *Полное решение.*

Читаем слева направо:
1. `Документ` — переменная, хранит объект «документ».
2. `Документ.Организация` — свойство документа «Организация»; оно само является объектом.
3. `Документ.Организация.ИНН` — свойство объекта-организации «ИНН»; это уже «плоское» значение (строка с цифрами).

`Сообщить(...)` выводит это значение на экран.

---

**К упражнению 3.** *Полное решение.*

Рассуждение ошибочно. `.Количество()` — метод объекта `Массив`: он вызывается через точку *на объекте*. `СтрДлина()` — глобальная функция: она принимает строку как аргумент, но не является частью строки как объекта. В 1С строки не являются объектами с методами — для работы со строками используются глобальные функции.

Признак метода — точка перед именем. Признак глобальной функции — нет точки, есть аргумент в скобках.

---

**К упражнению 4.** *Полное решение.*

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Тетрадь");
Товары.Добавить("Ластик");

Сообщить("Количество: " + Товары.Количество());
Сообщить("Второй элемент: " + Товары.Получить(1));
```

Экран:
```
Количество: 3
Второй элемент: Тетрадь
```

Нумерация с нуля: `Получить(0)` — «Ручка», `Получить(1)` — «Тетрадь», `Получить(2)` — «Ластик».

---

*Массив — вспомогательный объект, который мы создаём сами. Но в реальной конфигурации 1С есть объекты другого рода: справочники, документы. Они уже существуют в базе данных — их не создают через `Новый`, к ним обращаются. Именно это разбирается в § 5.2.*
