## § 5.4. Что такое ссылка и зачем она нужна

В предыдущих параграфах слово «ссылка» встречалось уже несколько раз: метод `НайтиПоНаименованию` возвращает ссылку, реквизит `Контрагент` является ссылкой, `Строка.Номенклатура` — ссылка. Пора разобраться, что это такое и почему 1С хранит именно ссылки, а не сами объекты.

---

## 1. Книга и адрес книги

Представьте библиотеку. Когда читатель хочет найти книгу, он не несёт её с собой повсюду — он записывает шифр: «зал 3, полка 7, место 12». Шифр — не сама книга, но с его помощью можно найти и прочитать любую её страницу.

В 1С устроено так же. База данных — это «библиотека». Объекты — документы, элементы справочников — хранятся на «полках» (в таблицах базы данных). **Ссылка** — это шифр: небольшое значение, которое однозначно указывает на конкретный объект.

```
  Ссылка ≠ Объект
  ──────────────────────────────────────────────────
  Ссылка:  «документ РТ-000042»          (маленькая, лёгкая)
  Объект:  весь документ с реквизитами   (может быть большим)
           и табличной частью
```

> **Ссылка — это указатель на объект в базе данных. Она не содержит данных объекта, но позволяет к ним обратиться.**

---

## 2. Зачем хранить ссылки, а не объекты

Когда в документе «Реализация» стоит поле «Контрагент», там не хранится вся карточка контрагента — его название, адрес, ИНН, история платежей. Хранится только ссылка: «вот этот конкретный контрагент из справочника».

Это правильное решение по нескольким причинам:

- **Экономия:** ссылка занимает несколько байт. Полная карточка контрагента — сотни байт и больше.
- **Актуальность:** если у контрагента изменился адрес, исправляют один раз — в справочнике. Все тысячи документов автоматически «видят» новый адрес, потому что хранят ссылку, а не копию.
- **Целостность:** нельзя написать в документе «контрагент Рога и копыта», если такого нет в справочнике. Ссылка всегда указывает на реальный объект.

> **Ссылка обеспечивает связь между объектами. Изменение данных в одном месте отражается везде, где на него есть ссылки.**

---

## 3. Тип ссылки

Ссылки бывают разных типов — по виду объекта, на который они указывают:

```
СправочникСсылка.Номенклатура     // ссылка на элемент справочника Номенклатура
СправочникСсылка.Контрагенты      // ссылка на элемент справочника Контрагенты
ДокументСсылка.РеализацияТоваров  // ссылка на документ РеализацияТоваров
```

В коде тип ссылки обычно не пишут явно — он определяется контекстом. Но понимать, что ссылки бывают разных видов, важно: нельзя передать ссылку на документ туда, где ожидается ссылка на контрагента.

---

## 4. Чтение данных через ссылку

Ссылка позволяет читать реквизиты объекта напрямую, через точку — без явного получения объекта:

```
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию("Ручка");

Сообщить(Ссылка.Наименование);   // читаем реквизит напрямую
Сообщить(Ссылка.Артикул);
```

Это работает для большинства реквизитов. При обращении к реквизиту через ссылку платформа сама делает запрос к базе данных — незаметно для программиста.

Однако в некоторых случаях нужен именно объект, а не ссылка. Например, чтобы добавить строку в табличную часть или записать изменения. Тогда используют метод `.ПолучитьОбъект()`:

```
Объект = Ссылка.ПолучитьОбъект();
Объект.Комментарий = "Актуально";
Объект.Записать();
```

> **Через ссылку можно читать реквизиты напрямую. Для изменения объекта нужен сам объект — его получают через `.ПолучитьОбъект()`.**

---

## 5. Пустая ссылка

Когда поиск не находит ничего, метод возвращает **пустую ссылку** — специальное значение, означающее «нет объекта». Пустая ссылка не указывает ни на какой объект; обращение к её реквизитам вызовет ошибку.

Проверить, пустая ли ссылка, можно двумя способами:

```
// Способ 1: через ЗначениеЗаполнено
Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить("Не найдено");
КонецЕсли;

// Способ 2: через метод ПустаяСсылка
Если Ссылка.ПустаяСсылка() Тогда
    Сообщить("Не найдено");
КонецЕсли;
```

Оба варианта эквивалентны. В реальном коде чаще встречается первый — `ЗначениеЗаполнено()` работает не только со ссылками, поэтому привычнее.

---

## 6. Ссылка как значение

Ссылку можно сохранить в переменную, передать в функцию, записать в реквизит другого объекта. Она ведёт себя как обычное значение — с одним отличием: это «адрес», а не «содержимое».

```
// Сохранить ссылку в переменную
КонтрагентСсылка = Справочники.Контрагенты.НайтиПоНаименованию("ООО Ромашка");

// Передать ссылку в реквизит документа
НовыйДокумент.Контрагент = КонтрагентСсылка;

// Сравнить две ссылки
Если СсылкаА = СсылкаБ Тогда
    Сообщить("Это один и тот же объект");
КонецЕсли;
```

Сравнение ссылок работает именно как сравнение адресов: две ссылки равны, если указывают на один и тот же объект.

---

## 7. Канцтовары: связь документов через ссылки

В системе «Канцтовары» накладная на реализацию связана с товарами через ссылки. Код, который выводит по каждой строке накладной полное наименование товара и его единицу измерения:

```
Ссылка = Документы.РеализацияТоваров.НайтиПоНомеру("РТ-000042");

Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить("Документ не найден");
    Возврат;
КонецЕсли;

Объект = Ссылка.ПолучитьОбъект();

Если Объект.Товары.Количество() = 0 Тогда
    Сообщить("Табличная часть пуста");
    Возврат;
КонецЕсли;

Для каждого Строка Из Объект.Товары Цикл
    Наименование = Строка.Номенклатура.Наименование;
    ЕдИзм = Строка.Номенклатура.ЕдиницаИзмерения.Наименование;
    Сообщить(Наименование + ", " + ЕдИзм + ": " + Строка.Количество);
КонецЦикла;
```

Разберём цепочку `Строка.Номенклатура.ЕдиницаИзмерения.Наименование`:
- `Строка.Номенклатура` — ссылка на элемент справочника Номенклатура.
- `.ЕдиницаИзмерения` — реквизит этого элемента, ссылка на справочник единиц измерения.
- `.Наименование` — строковый реквизит единицы измерения.

Три объекта, связанных ссылками, — и всё читается в одну строку.

---

## Главное в одном абзаце

Ссылка — это указатель на объект в базе данных. Она маленькая (несколько байт) и не содержит данных объекта — только адрес. Хранение ссылок вместо копий объектов обеспечивает актуальность данных: изменение в одном месте отражается везде. Через ссылку можно читать реквизиты напрямую через точку; для изменения объекта нужен сам объект — его получают через `.ПолучитьОбъект()`. Пустая ссылка означает «не найдено»; проверяется через `ЗначениеЗаполнено()` или `.ПустаяСсылка()`. Ссылки сравниваются как адреса: равны, если указывают на один и тот же объект.

## Упражнения

**Упражнение 1.** Объясните, что делает эта цепочка, шаг за шагом:

```
Сообщить(Документ.Склад.Ответственный.Наименование);
```

Сколько объектов участвует в цепочке? Что такое `Склад`, `Ответственный`, `Наименование`?

---

**Упражнение 2.** Найдите ошибку в коде:

```
Ссылка = Справочники.Контрагенты.НайтиПоНаименованию("Несуществующий");
Объект = Ссылка.ПолучитьОбъект();
Сообщить(Объект.ИНН);
```

Что произойдёт? Как исправить?

---

**Упражнение 3.** Когда нужен метод `.ПолучитьОбъект()`, а когда можно обойтись ссылкой? Приведите по одному примеру каждого случая.

---

**Упражнение 4.** В «Канцтоварах» нужно найти документ реализации, у которого `Контрагент.Наименование = "ООО Ромашка"`, и вывести сумму документа. Перебирайте все документы через `Выбрать()` и цикл `Пока`. Сумму ищите в реквизите шапки `СуммаДокумента`.

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Три объекта в цепочке:
1. `Документ` — сам документ.
2. `Документ.Склад` — реквизит документа, ссылка на склад (элемент справочника «Склады»).
3. `Документ.Склад.Ответственный` — реквизит склада, ссылка на сотрудника (элемент справочника «Сотрудники» или «Физические лица»).
4. `Документ.Склад.Ответственный.Наименование` — строковый реквизит сотрудника.

Цепочка читается: «у документа возьми склад, у склада возьми ответственного, у ответственного возьми наименование».

---

**К упражнению 2.** *Полное решение.*

Если «Несуществующий» не найден, `НайтиПоНаименованию` вернёт пустую ссылку. Вызов `.ПолучитьОбъект()` на пустой ссылке либо вернёт `Неопределено`, либо вызовет ошибку. Обращение к `.ИНН` пустого/неопределённого значения — ошибка выполнения.

Исправление — сторожевая проверка:

```
Ссылка = Справочники.Контрагенты.НайтиПоНаименованию("Несуществующий");
Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Сообщить("Контрагент не найден");
    Возврат;
КонецЕсли;
Объект = Ссылка.ПолучитьОбъект();
Сообщить(Объект.ИНН);
```

---

**К упражнению 3.** *Полное решение.*

Ссылка достаточна, если нужно только *читать* реквизиты:

```
Сообщить(Ссылка.Наименование);   // чтение — ссылки достаточно
```

Объект нужен, если нужно *изменить* данные или работать с табличной частью:

```
Объект = Ссылка.ПолучитьОбъект();
Объект.Комментарий = "Проверено"; // запись — нужен объект
Объект.Записать();
```

---

**К упражнению 4.** *Указание.*

Используйте цикл `Пока Выборка.Следующий()` (§ 5.2). Внутри цикла проверьте `Выборка.Контрагент.Наименование = "ООО Ромашка"`. Если совпало — выведите `Выборка.СуммаДокумента` и выйдите (`Прервать` или `Возврат`). Это паттерн поиска (§ 4.2).
