## § 5.5. Типичные обороты реального кода

За пять параграфов мы освоили понятийный каркас: объекты, свойства, методы, менеджеры, ссылки, табличные части. Теперь — каталог. В реальном коде 1С одни и те же обороты встречаются снова и снова. Узнать их с первого взгляда — значит читать код значительно быстрее. Этот параграф короче предыдущих намеренно: здесь не новая теория, а инвентарь для запоминания.

---

## 1. Проверка заполненности: `ЗначениеЗаполнено()`

Мы уже встречали эту функцию в § 5.2 и § 5.4. Самое время рассмотреть её полностью.

`ЗначениеЗаполнено(Значение)` возвращает `Истина`, если значение «не пустое». Что считается пустым:

```
  Тип              Пустое значение
  ─────────────    ────────────────────
  Число            0
  Строка           "" (пустая строка)
  Дата             01.01.0001 (пустая дата)
  Ссылка           пустая ссылка
  Булево           Ложь (считается пустым; булевые флаги обычно проверяют через НЕ напрямую)
```

Типичное использование — сторожевая проверка в начале процедуры:

```
Процедура ОбработатьТовар(ТоварСсылка)
    Если НЕ ЗначениеЗаполнено(ТоварСсылка) Тогда
        Сообщить("Товар не указан");
        Возврат;
    КонецЕсли;
    // основная логика
КонецПроцедуры;
```

> **`ЗначениеЗаполнено()` — универсальная проверка «есть ли реальное значение». Одна из самых частых функций в коде 1С.**

---

## 2. Получение и запись объекта

Стандартный цикл «найти → изменить → сохранить»:

```
Ссылка = Справочники.Номенклатура.НайтиПоНаименованию("Ручка");

Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
    Возврат;
КонецЕсли;

Объект = Ссылка.ПолучитьОбъект();   // получить объект для изменения
Объект.Комментарий = "Обновлено";    // изменить реквизит
Объект.Записать();                   // сохранить в базу
```

Метод `.Записать()` — процедура, которая сохраняет все изменения объекта в базу данных. Без вызова `.Записать()` изменения останутся только в оперативной памяти и будут утеряны при завершении программы.

> **Цикл «получить объект → изменить → записать» — базовый приём работы с данными.**

---

## 3. Перебор выборки: `Пока Выборка.Следующий() Цикл`

Этот оборот мы видели в § 5.2. Он настолько частый, что стоит выделить его отдельно:

```
Выборка = Справочники.Номенклатура.Выбрать();
Пока Выборка.Следующий() Цикл
    Сообщить(Выборка.Наименование);
КонецЦикла;
```

Внутри тела цикла `Выборка` ведёт себя как объект текущей записи: её реквизиты читаются через точку напрямую. Это удобнее, чем сначала получать ссылку, потом объект.

Выборку можно получить и с отбором — например, только проведённые документы или только элементы определённой группы. Условия отбора передаются в метод `.Выбрать()` аргументами; детали — в документации платформы.

---

## 4. Создание нового объекта

Иногда нужно не найти существующий объект, а создать новый. Для этого у менеджера есть метод `.СоздатьЭлемент()` (для справочников) или `.СоздатьДокумент()` (для документов):

```
// Создать новый элемент справочника
НовыйТовар = Справочники.Номенклатура.СоздатьЭлемент();
НовыйТовар.Наименование = "Степлер";
НовыйТовар.Артикул = "ST-001";
НовыйТовар.Записать();

// Создать новый документ
НовыйДокумент = Документы.РеализацияТоваров.СоздатьДокумент();
НовыйДокумент.Дата = ТекущаяДата();
НовыйДокумент.Контрагент = Справочники.Контрагенты.НайтиПоНаименованию("ООО Ромашка");
НовыйДокумент.Записать();
```

Объект создаётся в оперативной памяти и записывается в базу только после `.Записать()`.

---

## 5. Проверка реквизита-ссылки напрямую

Когда реквизит является ссылкой, его часто проверяют через `ЗначениеЗаполнено()` прямо в условии:

```
Если ЗначениеЗаполнено(Документ.Склад) Тогда
    Сообщить("Склад: " + Документ.Склад.Наименование);
Иначе
    Сообщить("Склад не указан");
КонецЕсли;
```

Это короткий и понятный оборот: «если склад указан — вывести его название, иначе — сообщить об отсутствии».

---

## 6. Функция `ТекущаяДата()`

Встроенная функция `ТекущаяДата()` возвращает текущие дату и время. Используется при создании новых документов и записей:

```
НовыйДокумент.Дата = ТекущаяДата();
```

Это глобальная функция (без точки и объекта), как `СтрДлина()` из § 5.1.

---

## 7. Метод `.Найти()` у массива и списка значений

В § 5.1 упоминались и другие методы массива. Один из частых — `.Найти(Значение)`: ищет значение в коллекции и возвращает индекс (порядковый номер с нуля), или `-1`, если не найдено.

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Карандаш");
Товары.Добавить("Тетрадь");

Индекс = Товары.Найти("Карандаш");
Если Индекс >= 0 Тогда
    Сообщить("Найден на позиции: " + Индекс);
Иначе
    Сообщить("Не найден");
КонецЕсли;
```

Это более короткая запись паттерна поиска (§ 4.2) — когда нужно только проверить наличие, без дополнительных условий.

---

## 8. Форматирование значений: `Формат()`

Встроенная функция `Формат(Значение, Параметры)` преобразует число или дату в строку с заданным форматом. Часто используется в сообщениях и отчётах:

```
Сообщить("Сумма: " + Формат(Итого, "ЧДЦ=2"));       // два знака после запятой
Сообщить("Дата: " + Формат(ТекущаяДата(), "ДФ=дд.ММ.гггг")); // дата без времени
```

Параметры форматирования записываются в виде строки с ключами. Точный синтаксис — в документации; пока главное знать, что функция `Формат()` существует и для чего она.

---

## 9. Канцтовары: сводная картина кода

Соберём вместе типичные обороты в одной программе — обработка прихода товаров:

```
// 1. Найти документ
СсылкаНаДокумент = Документы.ПоступлениеТоваров.НайтиПоНомеру("ПТ-000007");
Если НЕ ЗначениеЗаполнено(СсылкаНаДокумент) Тогда
    Сообщить("Документ не найден");
    Возврат;
КонецЕсли;

// 2. Получить объект
Документ = СсылкаНаДокумент.ПолучитьОбъект();

// 3. Проверить табличную часть
Если Документ.Товары.Количество() = 0 Тогда
    Сообщить("Нет товаров в документе");
    Возврат;
КонецЕсли;

// 4. Агрегировать данные
ИтогоСумма = 0;
Для каждого Строка Из Документ.Товары Цикл
    Если НЕ ЗначениеЗаполнено(Строка.Номенклатура) Тогда
        Продолжить; // пропустить строки без товара
    КонецЕсли;
    ИтогоСумма = ИтогоСумма + Строка.Сумма;
КонецЦикла;

// 5. Вывести результат
Сообщить("Итого по поступлению: " + Формат(ИтогоСумма, "ЧДЦ=2") + " руб.");
```

В одном фрагменте — защитный код, работа с выборкой, перебор табличной части, агрегация, `Продолжить` для фильтрации и форматированный вывод. Всё это уже знакомые инструменты.

---

## Главное в одном абзаце

Каталог устойчивых оборотов: `ЗначениеЗаполнено()` — универсальная проверка на «не пустое» для любых типов; цикл `Пока Выборка.Следующий()` — стандартный перебор всех записей вида; цикл «найти → `.ПолучитьОбъект()` → изменить → `.Записать()`» — базовый приём изменения данных; `.СоздатьЭлемент()` / `.СоздатьДокумент()` — создание новых записей; `Формат()` — форматирование числа или даты для вывода; `.Найти()` у массива — быстрая проверка наличия элемента. Эти обороты встречаются в любой конфигурации — теперь они узнаваемы.

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

(а) Сторожевая проверка: если реквизит `Контрагент` у документа не заполнен (пустая ссылка) — выйти из подпрограммы. Это защитный код перед работой с контрагентом.

(б) Получить объект по ссылке, изменить реквизит `Статус`, сохранить изменения в базу.

(в) Перебрать все элементы справочника «Склады» и вывести наименование каждого. Паттерн перебора, применённый к выборке из базы.
