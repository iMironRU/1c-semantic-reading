## § 5.3. Табличные части: коллекция строк внутри документа

В § 5.2 мы читали реквизиты документа как отдельные поля: номер, дата, контрагент. Но любой кто хоть раз открывал накладную или счёт, знает: основное содержимое — это не одно поле, а таблица. Товар, количество, цена, сумма — и таких строк может быть десятки. Это **табличная часть**, и работать с ней — один из самых частых навыков при чтении кода 1С.

---

## 1. Что такое табличная часть

Табличная часть — это коллекция строк, встроенная в объект. Не отдельная переменная, а именно часть документа или справочника: без него она не существует.

Каждая строка табличной части — сама по себе объект. У неё есть свои реквизиты — колонки таблицы.

```
  Документ «РеализацияТоваров»
  ┌────────────────────────────────────────────┐
  │ Номер: РТ-000042                           │
  │ Дата: 15.05.2026                           │
  │ Контрагент: ООО Ромашка                    │
  │                                            │
  │ Табличная часть «Товары»:                  │
  │ ┌──────────────┬────┬───────┬──────────┐   │
  │ │Номенклатура  │Кол │Цена   │Сумма     │   │
  │ ├──────────────┼────┼───────┼──────────┤   │
  │ │Ручка         │10  │35.00  │350.00    │   │
  │ │Тетрадь       │5   │60.00  │300.00    │   │
  │ │Карандаш      │20  │15.00  │300.00    │   │
  │ └──────────────┴────┴───────┴──────────┘   │
  └────────────────────────────────────────────┘
```

Реквизиты шапки (`Номер`, `Дата`, `Контрагент`) — это «плоские» свойства объекта. Табличная часть (`Товары`) — это коллекция, хранящаяся в объекте как одно из его свойств.

---

## 2. Как обратиться к табличной части

Табличная часть доступна через точку — как любое свойство объекта:

```
Объект.Товары          // коллекция строк табличной части
Объект.Услуги          // другая табличная часть того же документа
```

Коллекция строк имеет метод `.Количество()` — точно как массив:

```
Сообщить(Объект.Товары.Количество()); // сколько строк в табличной части
```

> **Табличная часть — свойство объекта, хранящее коллекцию строк. Обращаются через точку: `Объект.ИмяТабличнойЧасти`. Доступны те же методы, что у любой коллекции.**

---

## 3. Перебор строк табличной части

Строки табличной части перебираются циклом `Для каждого` — точно как элементы массива (§ 3.3, § 4.3):

```
Для каждого Строка Из Объект.Товары Цикл
    Сообщить(Строка.Номенклатура.Наименование);
    Сообщить(Строка.Количество);
    Сообщить(Строка.Цена);
    Сообщить(Строка.Сумма);
КонецЦикла;
```

Переменная цикла `Строка` на каждой итерации получает очередную строку табличной части. Строка — объект; её реквизиты читаются через точку. `Строка.Номенклатура` — это ссылка на элемент справочника, поэтому дальше стоит `.Наименование`.

Это паттерн перебора (§ 4.3), применённый к новому виду коллекции.

---

## 4. Агрегация по табличной части

Паттерны из Модуля 4 работают с табличной частью без изменений. Посчитать общую сумму строк:

```
ИтогоСумма = 0;
Для каждого Строка Из Объект.Товары Цикл
    ИтогоСумма = ИтогоСумма + Строка.Сумма;
КонецЦикла;
Сообщить("Итого: " + ИтогоСумма + " руб.");
```

Это агрегация (§ 4.4) — паттерн суммы. Коллекция другая, но форма та же.

Или найти строку с максимальной ценой — паттерн поиска (§ 4.2):

```
СтрокаМаксимума = Неопределено;
Для каждого Строка Из Объект.Товары Цикл
    Если СтрокаМаксимума = Неопределено Тогда
        СтрокаМаксимума = Строка;
    ИначеЕсли Строка.Цена > СтрокаМаксимума.Цена Тогда
        СтрокаМаксимума = Строка;
    КонецЕсли;
КонецЦикла;

Если ЗначениеЗаполнено(СтрокаМаксимума) Тогда
    Сообщить("Самая дорогая позиция: " + СтрокаМаксимума.Номенклатура.Наименование);
КонецЕсли;
```

Обратите внимание: переменная `СтрокаМаксимума` хранит целую строку табличной части, а не просто число. После цикла мы читаем реквизиты этой строки через точку.

> **Паттерны поиска, перебора и агрегации работают с табличными частями без изменений — табличная часть является коллекцией.**

---

## 5. Проверка перед перебором

Табличная часть может оказаться пустой — документ создан, но строки ещё не добавлены. Это та же ситуация, что с пустым массивом в § 4.5.

```
Если Объект.Товары.Количество() = 0 Тогда
    Сообщить("Табличная часть пуста — нечего обрабатывать");
    Возврат;
КонецЕсли;

// здесь знаем: строки есть
Для каждого Строка Из Объект.Товары Цикл
    ...
КонецЦикла;
```

Сторожевая проверка перед циклом — инвариант: после неё мы знаем, что коллекция непуста.

---

## 6. Добавление строк в табличную часть

Когда пишут код, создающий или заполняющий документ, строки добавляют методом `.Добавить()`. Добавленная строка возвращается как объект — к ней сразу можно обращаться:

```
НоваяСтрока = Объект.Товары.Добавить();
НоваяСтрока.Номенклатура = Справочники.Номенклатура.НайтиПоНаименованию("Ручка");
НоваяСтрока.Количество = 10;
НоваяСтрока.Цена = 35;
НоваяСтрока.Сумма = НоваяСтрока.Количество * НоваяСтрока.Цена;
```

Сначала — добавить строку и сохранить её в переменную. Потом — заполнить реквизиты строки через точку. Это типичный приём заполнения табличных частей в реальном коде.

---

## 7. Канцтовары: финансовый отчёт по реализации

Программа получает объект документа реализации и выводит полный отчёт:

```
// Предполагаем, что Документ уже получен (как — в § 5.4)
Если Документ.Товары.Количество() = 0 Тогда
    Сообщить("Документ без товаров");
    Возврат;
КонецЕсли;

ИтогоСумма = 0;
МаксЦена = 0;
МаксНазвание = "";

Для каждого Строка Из Документ.Товары Цикл
    ИтогоСумма = ИтогоСумма + Строка.Сумма;
    Если Строка.Цена > МаксЦена Тогда
        МаксЦена = Строка.Цена;
        МаксНазвание = Строка.Номенклатура.Наименование;
    КонецЕсли;
КонецЦикла;

Сообщить("Итого по документу: " + ИтогоСумма + " руб.");
Сообщить("Самый дорогой товар: " + МаксНазвание + " (" + МаксЦена + " руб.)");
```

Три паттерна в одном цикле: защитный код перед циклом, агрегация (сумма) и агрегация (максимум) одновременно внутри цикла. Форма — та же, что в § 4.4, только данные теперь берутся из настоящего документа.

---

## Главное в одном абзаце

Табличная часть — это коллекция строк, встроенная в документ или справочник как одно из его свойств. Обращаются к ней через точку: `Объект.ИмяТабличнойЧасти`. Каждая строка — сам по себе объект с реквизитами, доступными через точку. Перебор строк делается циклом `Для каждого`, агрегация и поиск — теми же паттернами, что в Модуле 4. Перед перебором непустоту табличной части стоит проверить через `.Количество()`. Новые строки добавляются методом `.Добавить()`, который возвращает добавленную строку — к ней сразу можно записывать реквизиты.

## Упражнения

**Упражнение 1.** Прочитайте код и объясните на русском языке, что он делает:

```
ПоложительныхСтрок = 0;
Для каждого Строка Из Документ.Товары Цикл
    Если Строка.Количество > 0 Тогда
        ПоложительныхСтрок = ПоложительныхСтрок + 1;
    КонецЕсли;
КонецЦикла;
Сообщить("Строк с ненулевым количеством: " + ПоложительныхСтрок);
```

---

**Упражнение 2.** Напишите программу, которая выводит наименование и сумму каждой строки табличной части `Товары` документа. Данных нет — считайте, что объект уже хранится в переменной `Объект`.

---

**Упражнение 3.** Напишите фрагмент, который добавляет в табличную часть `Услуги` документа новую строку: услуга «Доставка», количество 1, цена 500, сумма = количество × цена.

---

**Упражнение 4.** В «Канцтоварах» нужно проверить: есть ли в табличной части `Товары` хотя бы одна строка, у которой `Скидка > 0`. Напишите программу, которая выводит «Есть позиции со скидкой» или «Скидок нет». Используйте паттерн поиска.

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Программа считает, сколько строк в табличной части `Товары` имеют ненулевое количество (больше 0). Это паттерн агрегации — счётчик с условием (§ 4.4).

---

**К упражнению 2.** *Полное решение.*

```
Для каждого Строка Из Объект.Товары Цикл
    Сообщить(Строка.Номенклатура.Наименование + ": " + Строка.Сумма + " руб.");
КонецЦикла;
```

Если в `Объект.Товары` нет строк — цикл просто не выполнится ни разу. Для учебного примера это допустимо; в реальном коде стоит добавить проверку пустоты.

---

**К упражнению 3.** *Полное решение.*

```
НоваяСтрока = Объект.Услуги.Добавить();
НоваяСтрока.Номенклатура = Справочники.Номенклатура.НайтиПоНаименованию("Доставка");
НоваяСтрока.Количество = 1;
НоваяСтрока.Цена = 500;
НоваяСтрока.Сумма = НоваяСтрока.Количество * НоваяСтрока.Цена;
```

Метод `.Добавить()` возвращает только что добавленную строку. К ней сразу заполняем реквизиты.

---

**К упражнению 4.** *Указание.*

Используйте паттерн поиска (§ 4.2): инициализировать `НайденаСкидка = Ложь` (или через `Неопределено`), пройти по строкам, при условии `Строка.Скидка > 0` установить флаг и выйти через `Прервать`. После цикла проверить флаг.

---

*Табличная часть — коллекция, живущая внутри документа. Мы умеем её перебирать и применять к ней знакомые паттерны. Но в коде мы постоянно встречаем «ссылку» — сначала на объект менеджера, потом на контрагента документа, потом на номенклатуру в строке. Что именно хранит ссылка и почему 1С устроена именно так — тема § 5.4.*
