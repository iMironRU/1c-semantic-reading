## § 5.6. Первое чтение фрагмента из настоящей конфигурации

В § 4.6 мы учились распознавать паттерны в незнакомом коде. Тогда этот код был учебным — аккуратным и простым. Настоящий код выглядит иначе: имена переменных реальные, в одном фрагменте несколько паттернов, встречаются 1С-специфичные обороты. Именно это нас теперь и ждёт.

Этот параграф — не новая теория. Это тренировка чтения. Три фрагмента: от простого к сложному. К каждому — разбор по шагам.

---

## 1. Как читать незнакомый фрагмент

Алгоритм из § 4.6 остаётся прежним — просто добавился новый слой:

**Шаг 1: Какие конструкции?**
Цикл? Условие? Вызов метода? Работа с объектом через точку?

**Шаг 2: Какая форма?**
Распознать паттерн (поиск / перебор / агрегация / защитный код) *и* 1С-идиому (менеджер / выборка / табличная часть / ссылка).

**Шаг 3: Что именно делается?**
С какими данными? Какова бизнес-цель фрагмента?

---

## 2. Фрагмент первый: простой

```
Процедура ПроверитьНаличиеТовара(НаименованиеТовара)
    Ссылка = Справочники.Номенклатура.НайтиПоНаименованию(НаименованиеТовара);
    Если НЕ ЗначениеЗаполнено(Ссылка) Тогда
        Сообщить("Товар «" + НаименованиеТовара + "» не найден в справочнике");
        Возврат;
    КонецЕсли;
    Сообщить("Товар найден: " + Ссылка.Наименование + " (арт. " + Ссылка.Артикул + ")");
КонецПроцедуры;
```

*Шаг 1:* процедура с одним параметром. Внутри — вызов метода менеджера, сторожевая проверка с ранним `Возврат`, чтение реквизитов через точку.

*Шаг 2:* **защитный код** (сторожевая проверка результата поиска) + **идиома менеджера** (поиск по наименованию в справочнике).

*Шаг 3:* процедура принимает название товара, ищет его в справочнике Номенклатура. Если не нашла — сообщает об этом и выходит. Если нашла — выводит наименование и артикул.

Это весь фрагмент. Ничего загадочного: знакомый защитный код, только объект — не массив, а справочник.

---

## 3. Фрагмент второй: средний

```
Функция РассчитатьИтогДокумента(ДокументСсылка)
    Если НЕ ЗначениеЗаполнено(ДокументСсылка) Тогда
        Возврат 0;
    КонецЕсли;

    Объект = ДокументСсылка.ПолучитьОбъект();

    Если Объект.Товары.Количество() = 0 Тогда
        Возврат 0;
    КонецЕсли;

    ИтогоСумма = 0;
    Для каждого Строка Из Объект.Товары Цикл
        Если НЕ ЗначениеЗаполнено(Строка.Номенклатура) Тогда
            Продолжить;
        КонецЕсли;
        ИтогоСумма = ИтогоСумма + Строка.Количество * Строка.Цена;
    КонецЦикла;

    Возврат ИтогоСумма;
КонецФункции;
```

*Шаг 1:* функция с параметром-ссылкой. Две сторожевых проверки, получение объекта, цикл по табличной части, `Продолжить` для пропуска, накопление суммы, возврат.

*Шаг 2:* **защитный код** (две сторожевых проверки: пустая ссылка + пустая таблица) + **агрегация** (сумма) + **перебор с фильтрацией** (`Продолжить` для строк без товара). Идиомы: `.ПолучитьОбъект()`, обращение к табличной части, `ЗначениеЗаполнено()`.

*Шаг 3:* функция принимает ссылку на документ и возвращает итоговую сумму строк табличной части `Товары`. Сумма считается как `Количество × Цена` по каждой строке. Строки без заполненной номенклатуры пропускаются. Если документ не передан или таблица пустая — возвращает ноль.

Обратите внимание: в документе может быть реквизит `СуммаДокумента`, который хранит итог. Но эта функция считает сумму сама — вероятно, для проверки или для случая, когда автоматический подсчёт не сработал.

## 4. Фрагмент третий: развёрнутый

```
Процедура СформироватьОтчётПоПродажам(ДатаНачала, ДатаОкончания)
    Если ДатаНачала > ДатаОкончания Тогда
        Сообщить("Дата начала не может быть позже даты окончания");
        Возврат;
    КонецЕсли;

    КоличествоДокументов = 0;
    ОбщаяСумма = 0;

    Выборка = Документы.РеализацияТоваров.Выбрать(ДатаНачала, ДатаОкончания);
    Пока Выборка.Следующий() Цикл
        Если НЕ Выборка.Проведен Тогда
            Продолжить;
        КонецЕсли;
        КоличествоДокументов = КоличествоДокументов + 1;
        ОбщаяСумма = ОбщаяСумма + Выборка.СуммаДокумента;
    КонецЦикла;

    Если КоличествоДокументов = 0 Тогда
        Сообщить("За период не найдено проведённых документов");
        Возврат;
    КонецЕсли;

    Сообщить("Период: " + Формат(ДатаНачала, "ДФ=дд.ММ.гггг") +
             " — " + Формат(ДатаОкончания, "ДФ=дд.ММ.гггг"));
    Сообщить("Проведённых документов: " + КоличествоДокументов);
    Сообщить("Общая сумма продаж: " + Формат(ОбщаяСумма, "ЧДЦ=2") + " руб.");
КонецПроцедуры;
```

*Шаг 1:* процедура с двумя параметрами-датами. Сторожевая проверка дат. Два накопителя. Выборка документов с параметрами периода. Цикл с `Пока`. `Продолжить` для непроведённых. Агрегация. Финальная проверка. Форматированный вывод.

*Шаг 2:* **защитный код** (проверка параметров) + **агрегация** (счётчик + сумма) + **перебор с фильтрацией** (только проведённые документы) + **защитный код** (проверка результата после цикла). Идиомы: `Выбрать(ДатаНачала, ДатаОкончания)` (выборка с периодом), `Выборка.Проведен`, `Формат()`.

*Шаг 3:* процедура формирует отчёт о продажах за период. Перебирает все документы реализации в указанном диапазоне дат. Непроведённые документы пропускает. Считает количество проведённых документов и их суммарную стоимость. Если ни одного проведённого документа нет — сообщает об этом. Иначе — выводит итоги с форматированием.

## 5. Что осталось за скобками

Прочитав три фрагмента, легко заметить: мы понимаем *что* делает каждый фрагмент, но не всегда понимаем *почему именно так*. Почему сумма считается через `Количество × Цена`, а не из реквизита `Сумма`? Почему выборка с датами именно такая? Что происходит с документом при проведении?

Это вопросы другого уровня — контекста. Код отвечает на вопрос «как». Контекст отвечает на вопрос «зачем» и «почему». Это тема Модуля 6.

Но первый шаг уже сделан: незнакомый фрагмент в 30 строк — читается. Не идеально, не без усилий, но читается. Это и есть щёлчок Модуля 5.

> **Чтение кода — это навык. Сначала читаешь медленно, с усилием. Потом — быстрее. Потом — узнаёшь знакомые контуры раньше, чем дочитываешь до конца.**

---

## Главное в одном абзаце

Чтение незнакомого реального кода строится на том же трёхшаговом алгоритме (§ 4.6), к которому теперь добавился новый слой — распознавание 1С-идиом. Менеджер + поиск + проверка `ЗначениеЗаполнено` — это один блок. `.ПолучитьОбъект()` + изменение + `.Записать()` — другой. Выборка с `Пока Выборка.Следующий()` + фильтрация через `Продолжить` — третий. В реальном коде все эти блоки сочетаются и накладываются друг на друга. Понимать отдельный фрагмент — значит называть каждый блок словом и объяснять, что он делает с какими данными. Контекст — зачем это вообще нужно — приходит из понимания бизнес-задачи, которую решает программа.

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Паттерны: **защитный код** (сторожевая проверка параметра) + **перебор с фильтрацией** (пропуск товаров других поставщиков) + внутри — цикл «получить → изменить → записать».

Идиомы: `ЗначениеЗаполнено()`, `Выбрать()`, `Пока Выборка.Следующий()`, `Выборка.Ссылка.ПолучитьОбъект()`, `.Записать()`.

Процедура принимает ссылку на поставщика и процент изменения цены. Перебирает весь справочник Номенклатура, пропуская товары, принадлежащие другим поставщикам. Для товаров указанного поставщика увеличивает цену на заданный процент и сохраняет изменения.

---

**К упражнению 2.** *Полное решение.*

Если даты совпадают, `ДатаНачала > ДатаОкончания` — ложь, проверка пропускается. Выборка за период, где начало и конец одинаковы, вернёт документы за этот конкретный день — это корректный сценарий.

Проверка `>` защищает от ситуации, когда начало позже конца (тогда период не имеет смысла). Одинаковые даты — допустимый случай (один день), который не требует защиты.

---

**К упражнению 3.** *Полное решение.*

```
Функция ПолучитьИНН(КонтрагентСсылка)
    Если НЕ ЗначениеЗаполнено(КонтрагентСсылка) Тогда
        Возврат "";
    КонецЕсли;
    Возврат КонтрагентСсылка.ИНН;
КонецФункции;
```

Сторожевая проверка выносит нештатный случай в начало. Основной код остаётся одной строкой.

---

**К упражнению 4.** *Полное решение.*

```
Процедура ВывестиТоварыДешевле(ПорогЦены)
    Если ПорогЦены <= 0 Тогда
        Сообщить("Порог цены должен быть больше нуля");
        Возврат;
    КонецЕсли;

    Выборка = Справочники.Номенклатура.Выбрать();
    Пока Выборка.Следующий() Цикл
        Если Выборка.Цена < ПорогЦены Тогда
            Сообщить(Выборка.Наименование + " — " + Выборка.Цена + " руб.");
        КонецЕсли;
    КонецЦикла;
КонецПроцедуры;
```

## Итог Модуля 5

Мы прошли путь от массива, созданного вручную, до чтения настоящих процедур конфигурации. Узловые точки:

- **§ 5.1** — объект: значение с внутренней структурой, точка как операция «загляни внутрь».
- **§ 5.2** — менеджеры справочников и документов: как найти запись в базе данных.
- **§ 5.3** — табличные части: коллекция строк внутри документа, паттерны Модуля 4 в новом контексте.
- **§ 5.4** — ссылка: указатель на объект, а не сам объект; почему 1С хранит именно ссылки.
- **§ 5.5** — каталог устойчивых оборотов: `ЗначениеЗаполнено`, `.ПолучитьОбъект()`, `.Записать()`, `Формат()`.
- **§ 5.6** — первое чтение настоящего кода.

Паттерны из Модуля 4 никуда не делись — они всё так же работают. Только коллекции теперь другие: не `Новый Массив`, а табличная часть документа или выборка из справочника.

В Модуле 6 мы сделаем следующий шаг: от «что делает код» — к «зачем этот код существует». Документы как события бизнеса, жизненный цикл проведения, связь кода с реальными задачами фирмы.
