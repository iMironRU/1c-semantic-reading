## § 4.6. Распознавание паттернов в незнакомом коде

Четыре паттерна разобраны: поиск, перебор, агрегация, защитный код. Теперь — главный навык этого модуля: видеть их в коде, который вы не писали сами.

Когда разработчик открывает чужой файл, он не читает каждую строку с нуля. Он смотрит на форму кода и узнаёт знакомые очертания: «вот это поиск», «вот это накопитель», «вот сторожевая проверка перед циклом». Понимание приходит раньше, чем дочитываешь до конца.

Этот параграф — тренировка именно этого взгляда.

---

## 1. Алгоритм распознавания

Когда встречаете незнакомый фрагмент кода, задайте себе три вопроса в порядке:

**Шаг 1: Что за конструкции здесь есть?**
Цикл? Условие? Функция? Вызовы подпрограмм? Инициализация переменных?

**Шаг 2: Какова форма?**
Есть ли переменная, инициализированная до цикла и обновляемая внутри? — агрегация.
Есть ли `Прервать` внутри цикла? — поиск.
Цикл идёт до конца без `Прервать`? — перебор.
Есть ли `Если` в начале, проверяющий пустоту или `Неопределено`? — защитный код.

**Шаг 3: Что именно делает этот паттерн здесь?**
Какие данные? Что ищем / обрабатываем / считаем?

> **Распознать паттерн — значит назвать его одним словом, а потом уточнить: что именно делается с какими данными.**

---

## 2. Пример: разбор незнакомого фрагмента

Вот фрагмент. Прочитайте его и попробуйте назвать паттерн до разбора.

```
Сумма = 0;
Н = 0;
Для каждого Значение Из Данные Цикл
    Если Значение > 0 Тогда
        Сумма = Сумма + Значение;
        Н = Н + 1;
    КонецЕсли;
КонецЦикла;

Если Н > 0 Тогда
    Сообщить("Среднее положительных: " + Сумма / Н);
Иначе
    Сообщить("Нет положительных значений");
КонецЕсли;
```

*Шаг 1:* цикл `Для каждого`, условие внутри, две переменные инициализированы до цикла, проверка после.

*Шаг 2:* два накопителя — сумма и счётчик — обновляются внутри цикла с условием. После цикла — защитная проверка знаменателя. Это **агрегация** (среднее) с **фильтрацией** (только положительные) и **защитным кодом** (деление).

*Шаг 3:* вычислить среднее арифметическое только положительных элементов коллекции. Если таких нет — сообщить об этом.

Три паттерна в одном фрагменте — это нормально. Паттерны сочетаются.

---

## 3. Сигнатуры паттернов

Каждый паттерн имеет характерные «сигнатуры» — черты, по которым его узнают с первого взгляда:

```
  ПОИСК
  ─────
  НайденноеЧто-то = Неопределено;  ← инициализация пустышкой
  Для каждого ... Цикл
      Если ... Тогда
          НайденноеЧто-то = ...;
          Прервать;              ← ключевой маркер
      КонецЕсли;
  КонецЦикла;
  Если НайденноеЧто-то = Неопределено Тогда  ← проверка
  
  ПЕРЕБОР
  ───────
  Для каждого ... Цикл
      // действие над каждым       ← нет Прервать
  КонецЦикла;
  
  АГРЕГАЦИЯ
  ─────────
  Накопитель = 0;                 ← инициализация перед циклом
  Для каждого ... Цикл
      Накопитель = Накопитель ... ← обновление через сам накопитель
  КонецЦикла;
  
  ЗАЩИТНЫЙ КОД
  ────────────
  Если Коллекция.Количество() = 0 Тогда  ← проверка ДО цикла
      ...
  КонецЕсли;
  // или:
  Если Результат = Неопределено Тогда    ← проверка ПОСЛЕ поиска
```

Запомните эти «силуэты». Их узнавание — ключевое умение уровня 3.

---

## 4. Паттерны встречаются вместе

В реальном коде паттерны редко существуют по одному. Типичные комбинации:

- Защитный код + перебор: сначала проверить, что коллекция не пуста, потом обойти её.
- Поиск + защитный код: найти элемент, потом проверить, что нашли, прежде чем использовать.
- Перебор + агрегация: обойти коллекцию и одновременно накапливать сумму.

Умение видеть, где заканчивается один паттерн и начинается другой, — признак уверенного чтения.

---

## 5. Когда паттерн не узнаётся

Иногда код не укладывается в знакомые формы. Это может означать:

- **Нестандартная запись.** Тот же паттерн, но переменные названы необычно или структура чуть иначе. Читайте медленно, по конструкциям.
- **Новый паттерн.** Задача, с которой вы ещё не встречались. В этом случае прогонка (§ 3.7) — ваш инструмент: шаг за шагом восстановите, что делает код.
- **Сложная комбинация.** Несколько паттернов переплетены. Попробуйте выделить блоки: что делает вот этот кусок с первой по десятую строку — и назовите.

> **Если паттерн не узнаётся — прогоняйте. Прогонка превращает незнакомый код в знакомый.**

## 6. Канцтовары: читаем чужой код

Перед вами фрагмент, который написал другой программист. Прочитайте и ответьте: что он делает?

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Тетрадь");
Товары.Добавить("Карандаш");
Товары.Добавить("Линейка");

Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);

ЦенаПорога = 30;
ДорогихТоваров = 0;
СуммаДорогих = 0;

Для каждого Цена Из Цены Цикл
    Если Цена > ЦенаПорога Тогда
        ДорогихТоваров = ДорогихТоваров + 1;
        СуммаДорогих = СуммаДорогих + Цена;
    КонецЕсли;
КонецЦикла;

Если ДорогихТоваров > 0 Тогда
    Сообщить("Дорогих позиций: " + ДорогихТоваров);
    Сообщить("На сумму: " + СуммаДорогих + " руб.");
Иначе
    Сообщить("Все товары дешевле " + ЦенаПорога + " руб.");
КонецЕсли;
```

*Разбор:*

- Два накопителя (`ДорогихТоваров`, `СуммаДорогих`) инициализированы до цикла. — **агрегация**.
- Внутри цикла — фильтрация по `Если Цена > ЦенаПорога`. — **перебор с фильтрацией**.
- После цикла — проверка `ДорогихТоваров > 0` перед использованием. — **защитный код**.

Один абзацем: *программа считает количество и суммарную стоимость товаров, цена которых выше порогового значения. Если таких нет — сообщает об этом.*

## Главное в одном абзаце

Распознавание паттернов — это чтение кода на уровне намерений: не «что написано строка за строкой», а «что здесь делается». Алгоритм: выявить конструкции → распознать форму → уточнить содержание. Каждый паттерн имеет сигнатуру: поиск — `Неопределено` + `Прервать` + проверка после; перебор — цикл до конца без `Прервать`; агрегация — накопитель до цикла и обновление внутри; защитный код — проверка перед действием. В реальном коде паттерны сочетаются — нужно уметь видеть, где кончается один и начинается другой. Когда паттерн не узнаётся — прогонка возвращает понимание.

## Упражнения

**Упражнение 1.** Прочитайте фрагмент. Назовите паттерн. Опишите одним предложением, что делает программа.

```
ПервоеЧётное = Неопределено;
Для каждого Н Из Числа Цикл
    Если Н / 2 = Цел(Н / 2) Тогда
        ПервоеЧётное = Н;
        Прервать;
    КонецЕсли;
КонецЦикла;

Если ПервоеЧётное = Неопределено Тогда
    Сообщить("Чётных нет");
Иначе
    Сообщить("Первое чётное: " + ПервоеЧётное);
КонецЕсли;
```

*(Функция `Цел(Х)` возвращает целую часть числа X, отбрасывая дробную.)*

**Упражнение 2.** Прочитайте фрагмент. Назовите паттерн(ы). Что делает программа?

Иначе
    ДлинноеСлово = "";

**Упражнение 3.** Прочитайте фрагмент. Назовите паттерн(ы) и укажите, где в коде есть ошибка или уязвимость.

```
ЦеныПоставщика = Новый Массив;
ЦеныПоставщика.Добавить(120);
ЦеныПоставщика.Добавить(85);
ЦеныПоставщика.Добавить(200);

Минимум = 0;
Для каждого Цена Из ЦеныПоставщика Цикл
    Если Цена < Минимум Тогда
        Минимум = Цена;
    КонецЕсли;
КонецЦикла;
Сообщить("Минимальная цена: " + Минимум);
```

**Упражнение 4.** Напишите функцию `НайтиСамоеДлинноеСлово(Слова)`, которая принимает массив строк и возвращает самую длинную из них. Используйте защитный код для пустого массива.

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Паттерн: **поиск**.

Сигнатуры: `ПервоеЧётное = Неопределено` до цикла; `Прервать` при выполнении условия; проверка `= Неопределено` после.

Программа ищет первое чётное число в массиве. Число чётное, если его деление на 2 без остатка совпадает с самим делением (проверка через `Цел`). Если чётных нет — сообщает; иначе — выводит найденное.

---

**К упражнению 2.** *Полное решение.*

Паттерны: **защитный код** (проверка `Количество() = 0` до основной логики) + **агрегация** (поиск максимального по длине — вариант «максимум»).

Программа находит самое длинное слово в списке. Перед этим защищается от пустого списка. Внутри — два накопителя: `МакДлина` (числовой максимум) и `ДлинноеСлово` (само слово). Инициализация `МакДлина = 0` работает, если все слова содержат хотя бы одну букву — что обычно верно для реального текста.

---

**К упражнению 3.** *Полное решение.*

Паттерн: **агрегация** (минимум).

Уязвимость: `Минимум = 0` как начальное значение. Все три цены (120, 85, 200) больше нуля, поэтому ни одна не пройдёт условие `Цена < Минимум`. После цикла `Минимум` останется `0` — но `0` нет среди цен поставщика. Программа выдаёт неверный результат.

Правильный подход: флаговая инициализация (как в § 4.4).

```
МинимумНайден = Ложь;
Минимум = 0;
Для каждого Цена Из ЦеныПоставщика Цикл
    Если НЕ МинимумНайден Тогда
        Минимум = Цена;
        МинимумНайден = Истина;
    ИначеЕсли Цена < Минимум Тогда
        Минимум = Цена;
    КонецЕсли;
КонецЦикла;
Сообщить("Минимальная цена: " + Минимум);
```

Результат: `85`.

---

**К упражнению 4.** *Полное решение.*

```
Функция НайтиСамоеДлинноеСлово(Слова)
    Если Слова.Количество() = 0 Тогда
        Возврат "";
    КонецЕсли;

    МаксДлина = 0;
    ДлинноеСлово = "";

    Для каждого Слово Из Слова Цикл
        Если СтрДлина(Слово) > МаксДлина Тогда
            МаксДлина = СтрДлина(Слово);
            ДлинноеСлово = Слово;
        КонецЕсли;
    КонецЦикла;

    Возврат ДлинноеСлово;
КонецФункции;
```

Защитный код — сторожевая проверка в начале. Инициализация `МаксДлина = 0` работает для непустых строк (длина всегда ≥ 1, то есть больше 0).

## Итог Модуля 4

Паттерны — это уровень между конструкциями и смыслом. Мы освоили четыре:

- **Поиск:** найти один элемент, остановиться.
- **Перебор:** обойти всю коллекцию, обработать каждый (или каждый подходящий).
- **Агрегация:** свернуть коллекцию к одному значению — сумме, максимуму, счётчику.
- **Защитный код:** проверить данные прежде, чем с ними работать.

В Модуле 5 мы поднимемся на следующий уровень: от универсальных паттернов — к идиомам, специфичным для платформы 1С. Там заговорят объекты, ссылки, табличные части. Кое-что окажется знакомым — потому что под 1С-специфичной оболочкой работают те же паттерны.
