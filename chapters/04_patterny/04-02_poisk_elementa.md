## § 4.2. Поиск элемента

Самая частая задача при работе с коллекциями — найти элемент. Есть ли в списке товаров «Ручка»? Какая у неё цена? Присутствует ли клиент в базе? Это всё один и тот же вопрос в разных формах: *есть ли в коллекции элемент, удовлетворяющий условию, и если есть — какой именно?*

Ни одна конструкция языка не отвечает на этот вопрос сама по себе. Ответ получается из комбинации: цикл + условие + способ остановиться + способ сообщить результат. Это и есть паттерн поиска.

---

## 1. Типичная форма паттерна

Паттерн поиска в его базовом виде:

```
НайденныйЭлемент = Неопределено;
Для каждого Элемент Из Коллекция Цикл
    Если <условие на Элемент> Тогда
        НайденныйЭлемент = Элемент;
        Прервать;
    КонецЕсли;
КонецЦикла;

Если НайденныйЭлемент = Неопределено Тогда
    // не нашли
Иначе
    // нашли, НайденныйЭлемент содержит результат
КонецЕсли;
```

Четыре части:

1. **Инициализация:** `НайденныйЭлемент = Неопределено` — начальное значение, означающее «пока не нашли».
2. **Цикл по коллекции:** проходим по всем элементам.
3. **Проверка условия:** если элемент подходит — сохраняем его и выходим из цикла (`Прервать`).
4. **Проверка результата:** после цикла смотрим, осталось ли значение `Неопределено`.

> **Паттерн поиска: инициализировать «пустышку», пройти по коллекции, при совпадении сохранить элемент и остановиться, после цикла проверить результат.**

---

## 2. Ключевое слово «Прервать»

`Прервать` — это команда немедленно выйти из цикла, не дожидаясь его естественного конца. Выполнение переходит к первой команде после `КонецЦикла`.

```
Для каждого Элемент Из Коллекция Цикл
    Если Элемент = ИскомоеЗначение Тогда
        // нашли — незачем продолжать
        Прервать;
    КонецЕсли;
КонецЦикла;
// сюда попадём либо после перебора всех, либо после Прервать
```

`Прервать` прерывает только *ближайший* цикл. Если циклы вложены, внешний продолжит работу.

Зачем `Прервать` в паттерне поиска? Без него цикл продолжит перебирать оставшиеся элементы, даже когда нужный уже найден. Это лишняя работа — и, если коллекция большая, заметная потеря скорости.

> **`Прервать` выходит из ближайшего цикла немедленно. В паттерне поиска — это сигнал «нашли, дальше незачем».**

Есть родственная команда `Продолжить` — она пропускает оставшуюся часть тела цикла и переходит к следующей итерации, не выходя из цикла. Она пригодится в § 4.3. Пока запомним: `Прервать` — выход, `Продолжить` — пропуск итерации.

---

## 3. «Неопределено» как сигнал «не нашли»

`Неопределено` — особое значение в 1С, означающее «нет значения» или «значение ещё не установлено». Мы видели его в § 2.2 как один из типов данных.

В паттерне поиска `Неопределено` играет роль «пустышки»: переменная инициализируется им перед циклом, а после цикла значение `Неопределено` означает «ничего не нашли». Любое другое значение означает «нашли».

```
НайденноеИмя = Неопределено;
// ... цикл поиска ...

Если НайденноеИмя = Неопределено Тогда
    Сообщить("Не найдено");
Иначе
    Сообщить("Найдено: " + НайденноеИмя);
КонецЕсли;
```

Это устойчивый приём: переменная «говорит» через своё значение — нашли или нет.

---

## 4. Прогонка

Разберём конкретный пример. Ищем «Карандаш» в списке из трёх товаров:

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Карандаш");
Товары.Добавить("Тетрадь");

НайденныйТовар = Неопределено;
Для каждого Товар Из Товары Цикл
    Если Товар = "Карандаш" Тогда
        НайденныйТовар = Товар;
        Прервать;
    КонецЕсли;
КонецЦикла;

Если НайденныйТовар = Неопределено Тогда
    Сообщить("Товар не найден");
Иначе
    Сообщить("Найден: " + НайденныйТовар);
КонецЕсли;
```

| Шаг | Команда                                   | Товар (тек.) | НайденныйТовар | Экран |
|-----|-------------------------------------------|--------------|----------------|-------|
| 1   | `НайденныйТовар = Неопределено`           | —            | Неопределено   |       |
| 2   | итерация 1: `Товар = "Ручка"`             | Ручка        | Неопределено   |       |
| 3   | `"Ручка" = "Карандаш"` → Ложь            | Ручка        | Неопределено   |       |
| 4   | итерация 2: `Товар = "Карандаш"`          | Карандаш     | Неопределено   |       |
| 5   | `"Карандаш" = "Карандаш"` → Истина       | Карандаш     | Неопределено   |       |
| 6   | `НайденныйТовар = "Карандаш"`             | Карандаш     | Карандаш       |       |
| 7   | `Прервать` → выход из цикла               | —            | Карандаш       |       |
| 8   | `НайденныйТовар = Неопределено` → Ложь   | —            | Карандаш       |       |
| 9   | `Сообщить("Найден: Карандаш")`            | —            | Карандаш       | Найден: Карандаш |

Третий элемент «Тетрадь» так и не проверялся: `Прервать` вывел нас из цикла на шаге 7.

---

## 5. Вариация: поиск по условию, а не по точному совпадению

Условие в паттерне поиска не обязательно проверяет точное равенство. Можно искать по любому выражению:

```
// Найти первый товар дороже 50 рублей
НайденнаяЦена = Неопределено;
Для каждого Цена Из Цены Цикл
    Если Цена > 50 Тогда
        НайденнаяЦена = Цена;
        Прервать;
    КонецЕсли;
КонецЦикла;
```

Или поиск среди строк по началу слова (функция `Найти` возвращает позицию подстроки или 0, если не нашла; `= 1` означает «подстрока начинается с первого символа» — это пример встроенной подпрограммы, мимоходом; вернёмся в § 5.5):

```
// Найти товар, название которого начинается с "Кар"
НайденныйТовар = Неопределено;
Для каждого Товар Из Товары Цикл
    Если Найти(Товар, "Кар") = 1 Тогда
        НайденныйТовар = Товар;
        Прервать;
    КонецЕсли;
КонецЦикла;
```

Условие меняется — структура паттерна остаётся.

---

## 6. Вариация: найти все, а не первый

Иногда нужно не остановиться на первом найденном, а собрать *все* подходящие элементы. Тогда `Прервать` убирается, а результат накапливается в отдельную коллекцию:

```
НайденныеТовары = Новый Массив;
Для каждого Товар Из Товары Цикл
    Если <условие> Тогда
        НайденныеТовары.Добавить(Товар);
    КонецЕсли;
КонецЦикла;
```

Это уже ближе к паттерну перебора (§ 4.3) с фильтрацией. Граница между «поиском первого» и «поиском всех» — в наличии `Прервать`. Один поиск — есть `Прервать`. Все подходящие — нет.

---

## 7. Канцтовары: есть ли товар на складе

Магазин хранит список доступных товаров. Кассир вводит название — программа проверяет наличие:

```
ДоступныеТовары = Новый Массив;
ДоступныеТовары.Добавить("Ручка");
ДоступныеТовары.Добавить("Тетрадь");
ДоступныеТовары.Добавить("Карандаш");
ДоступныеТовары.Добавить("Линейка");
ДоступныеТовары.Добавить("Ластик");

ЗапросТовара = "Линейка";
НайденТовар = Неопределено;

Для каждого Товар Из ДоступныеТовары Цикл
    Если Товар = ЗапросТовара Тогда
        НайденТовар = Товар;
        Прервать;
    КонецЕсли;
КонецЦикла;

Если НайденТовар = Неопределено Тогда
    Сообщить("Товар «" + ЗапросТовара + "» отсутствует");
Иначе
    Сообщить("Товар «" + НайденТовар + "» есть в наличии");
КонецЕсли;
```

На экране: `Товар «Линейка» есть в наличии`.

Обратите внимание: цикл остановится на четвёртом элементе («Линейка»), не дойдя до «Ластика». Пять элементов, но перебраны только четыре.

---

## Главное в одном абзаце

Паттерн поиска решает задачу «найти в коллекции элемент, удовлетворяющий условию». Типичная форма: инициализировать переменную результата значением `Неопределено`, пройти по коллекции циклом, при выполнении условия сохранить элемент и немедленно выйти из цикла командой `Прервать`, после цикла проверить, осталось ли значение `Неопределено`. `Прервать` прерывает ближайший цикл и не даёт перебирать лишние элементы. Условие поиска может быть любым — точное равенство, диапазон, сравнение строк; структура паттерна при этом не меняется. Если нужно найти не один, а все подходящие элементы — `Прервать` убирается, результат накапливается в новую коллекцию.

## Контрольные вопросы

1. Из каких четырёх частей состоит паттерн поиска?
2. Для чего нужна инициализация `НайденныйЭлемент = Неопределено` до цикла?
3. Что делает команда `Прервать`? Как она влияет на вложенные циклы?
4. Как после цикла понять, был ли элемент найден?
5. Чем отличается «найти первый» от «найти все»? Что меняется в структуре паттерна?

```
НайденнаяЦена = Неопределено;
Для каждого Цена Из Цены Цикл
```

**Упражнение 3.** Напишите поиск: найти в массиве `["Ручка", "Тетрадь", "Карандаш", "Линейка"]` первый товар, название которого длиннее 6 символов. Используйте встроенную функцию `СтрДлина(Строка)`, которая возвращает длину строки в символах.

---

**Упражнение 4.** В «Канцтоварах» нужно найти все товары из списка, цена которых не превышает 30 рублей. Напишите программу, которая собирает такие товары в отдельный массив и выводит их.

Данные: `["Ручка", "Тетрадь", "Карандаш", "Линейка", "Ластик"]`, цены: `[35, 60, 15, 25, 10]`.

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Прогонка: «Резинки» нет в коллекции. Цикл проходит все три элемента, условие ни разу не выполняется. `НайденныйТовар` не изменяется — остаётся `Неопределено`.

После цикла: `НайденныйТовар = Неопределено` → Истина → выполняется ветка «Не найдено».

Экран: `Не найдено`.

---

**К упражнению 2.** *Полное решение.*

Ошибка: присваивание `НайденнаяЦена = Цена` стоит *до* условия. Это значит, что при каждой итерации переменная получает текущее значение — независимо от того, подходит оно или нет. Если подходящая цена не найдена, `НайденнаяЦена` будет содержать *последний* элемент коллекции, а не `Неопределено`.

Исправление — переставить присваивание *внутрь* условия:

```
НайденнаяЦена = Неопределено;
Для каждого Цена Из Цены Цикл
    Если Цена > 100 Тогда
        НайденнаяЦена = Цена;
        Прервать;
    КонецЕсли;
КонецЦикла;
```

---

**К упражнению 3.** *Указание.*

Структура паттерна та же. Условие: `СтрДлина(Товар) > 6`. «Карандаш» имеет 8 символов — он первый подходящий. Проверьте прогонкой: «Ручка» — 5, «Тетрадь» — 7 (первый подходящий — «Тетрадь»).

*Уточнение:* «Тетрадь» — 7 символов (`Т`, `е`, `т`, `р`, `а`, `д`, `ь`), это больше 6. Она найдётся второй в цикле.

---

**К упражнению 4.** *Полное решение.*

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Тетрадь");
Товары.Добавить("Карандаш");
Товары.Добавить("Линейка");
Товары.Добавить("Ластик");

Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);
Цены.Добавить(10);
```

Задача: найти товары с ценой ≤ 30. Но здесь два параллельных массива — имена и цены. Чтобы связать их, нужно знать порядковый номер элемента. Это задача для числового цикла (`Для Н = 0 По ...`), который мы пока не разбирали. Пока упростим: работаем только с ценами.

```
ДешёвыеЦены = Новый Массив;
Для каждого Цена Из Цены Цикл
    Если Цена <= 30 Тогда
        ДешёвыеЦены.Добавить(Цена);
    КонецЕсли;
КонецЦикла;

Для каждого Цена Из ДешёвыеЦены Цикл
    Сообщить(Цена + " руб.");
КонецЦикла;
```

Экран: `15 руб.`, `25 руб.`, `10 руб.`.

Обратите внимание: `Прервать` здесь нет — мы собираем *все* подходящие элементы, а не только первый.

---

*Паттерн поиска находит один элемент и останавливается. Следующий паттерн — перебор — проходит по всей коллекции и что-то делает с каждым элементом. § 4.3.*
