## § 4.3. Перебор и обработка коллекции

Паттерн поиска (§ 4.2) находит один элемент и останавливается. Но часто задача другая: *обработать каждый элемент коллекции* — вывести, изменить, записать, отфильтровать. Здесь нет цели найти и остановиться. Здесь цель — дойти до конца.

Это паттерн перебора. Он самый простой по форме и самый частый по применению.

---

## 1. Типичная форма паттерна

```
Для каждого Элемент Из Коллекция Цикл
    // обработать Элемент
КонецЦикла;
```

Никаких специальных переменных до цикла, никакого `Прервать`, никакой проверки после. Просто цикл по всей коллекции — и внутри любое действие над каждым элементом.

> **Паттерн перебора: пройти по всей коллекции и что-то сделать с каждым элементом. Условие выхода — исчерпание коллекции.**

Кажется, что это просто цикл `Для каждого Из`, который мы уже знаем из § 3.3. Так и есть. Паттерн перебора — это именно он. Но теперь мы видим его не как «синтаксическую конструкцию», а как *намерение*: пройти всё.

---

## 2. Три вида обработки

Внутри цикла может быть разная «обработка». Рассмотрим три типичных варианта.

**Вывод.** Самый простой: показать каждый элемент.

```
Для каждого Товар Из Товары Цикл
    Сообщить(Товар);
КонецЦикла;
```

**Преобразование.** Что-то вычислить по каждому элементу.

```
Для каждого Цена Из Цены Цикл
    ЦенаСоСкидкой = Цена * 0.9;
    Сообщить(ЦенаСоСкидкой + " руб.");
КонецЦикла;
```

**Сохранение результата в новую коллекцию.** Трансформировать коллекцию в другую.

```
ЦеныСоСкидкой = Новый Массив;
Для каждого Цена Из Цены Цикл
    ЦеныСоСкидкой.Добавить(Цена * 0.9);
КонецЦикла;
```

Во всех трёх случаях форма цикла одна и та же. Меняется только то, что происходит внутри.

---

## 3. Перебор с фильтрацией

Частая вариация — обрабатывать не все элементы, а только те, что удовлетворяют условию. Внутри цикла добавляется `Если`:

```
Для каждого Элемент Из Коллекция Цикл
    Если <условие> Тогда
        // обработать Элемент
    КонецЕсли;
КонецЦикла;
```

Это не поиск — мы не останавливаемся на первом подходящем. Мы обходим всю коллекцию, но обрабатываем только избранных.

Пример: вывести только дорогие товары.

```
Для каждого Цена Из Цены Цикл
    Если Цена > 30 Тогда
        Сообщить(Цена + " руб. — дорогой товар");
    КонецЕсли;
КонецЦикла;
```

> **Перебор с фильтрацией: пройти всю коллекцию, обработать только элементы, удовлетворяющие условию. `Прервать` нет — цикл идёт до конца.**

Здесь пригождается уже упомянутая команда `Продолжить`. Она пропускает остаток тела цикла и переходит к следующей итерации. С ней фильтр можно записать «от обратного»:

```
Для каждого Цена Из Цены Цикл
    Если Цена <= 30 Тогда
        Продолжить; // пропустить дешёвые
    КонецЕсли;
    Сообщить(Цена + " руб. — дорогой товар");
КонецЦикла;
```

Оба варианта дают одинаковый результат. Первый — «обрабатываем, если подходит». Второй — «пропускаем, если не подходит». Выбор — дело стиля и читаемости.

---

## 4. Прогонка перебора с фильтрацией

Данные: `Цены = [35, 60, 15, 25, 10]`. Выводим цены выше 25.

```
Для каждого Цена Из Цены Цикл
    Если Цена > 25 Тогда
        Сообщить(Цена + " руб.");
    КонецЕсли;
КонецЦикла;
```

| Шаг | Цена | Проверка `Цена > 25` | Экран |
|-----|------|----------------------|-------|
| 1   | 35   | Истина               | 35 руб. |
| 2   | 60   | Истина               | 60 руб. |
| 3   | 15   | Ложь                 |       |
| 4   | 25   | Ложь                 |       |
| 5   | 10   | Ложь                 |       |

Цикл прошёл все пять элементов. Вывел два — те, что строго больше 25.

---

## 5. Как отличить перебор от поиска

Паттерны перебора и поиска похожи — оба используют `Для каждого Из`. Различие принципиальное:

```
  Паттерн поиска           Паттерн перебора
  ─────────────────────    ─────────────────────────
  ищем ОДИН элемент        обрабатываем ВСЕ (или всех подходящих)
  есть Прервать            нет Прервать
  есть переменная-результат нет (или накапливаем новую коллекцию)
  после цикла — проверка   после цикла — ничего особого
```

Вопрос для самодиагностики: «Нужно ли мне остановиться, когда нашёл?» Если да — поиск. Если нет — перебор.

---

## 6. Канцтовары: прайс-лист и отчёт

Программа выводит два отчёта. Первый — полный прайс-лист. Второй — только товары для школьников (дешевле 30 рублей).

```
Товары = Новый Массив;
Товары.Добавить("Ручка");
Товары.Добавить("Тетрадь");
Товары.Добавить("Карандаш");
Товары.Добавить("Линейка");
Товары.Добавить("Ластик");

Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);
Цены.Добавить(10);

// Полный прайс: все товары
Сообщить("--- Все товары ---");
Для каждого Товар Из Товары Цикл
    Сообщить(Товар);
КонецЦикла;

// Дешёвые цены: только ≤ 30 рублей
Сообщить("--- До 30 руб. ---");
Для каждого Цена Из Цены Цикл
    Если Цена <= 30 Тогда
        Сообщить(Цена + " руб.");
    КонецЕсли;
КонецЦикла;
```

На экране:

```
--- Все товары ---
Ручка
Тетрадь
Карандаш
Линейка
Ластик
--- До 30 руб. ---
15 руб.
25 руб.
10 руб.
```

Два прохода по двум коллекциям, два паттерна перебора: один без фильтра, другой с фильтром.

---

## Главное в одном абзаце

Паттерн перебора: пройти по всей коллекции и что-то сделать с каждым элементом. Отличается от поиска тем, что цель — не найти и остановиться, а обойти всё; поэтому `Прервать` отсутствует. Внутри цикла может быть вывод, вычисление, сохранение в другую коллекцию — что угодно. Перебор с фильтрацией добавляет `Если` внутрь цикла: обрабатываются только элементы, удовлетворяющие условию; `Продолжить` позволяет записать фильтр «от обратного», пропуская не подходящие элементы. Вопрос-маркер: «нужно ли остановиться после первого найденного?» — если нет, это перебор.

## Упражнения

**Упражнение 1.** Прочитайте код и назовите паттерн: поиск или перебор? Почему?

```
Для каждого Товар Из Товары Цикл
    Если СтрДлина(Товар) > 5 Тогда
        Сообщить(Товар);
    КонецЕсли;
КонецЦикла;
```

---

**Упражнение 2.** Напишите программу: вывести все цены из массива `[35, 60, 15, 25, 10]`, умноженные на 1.2 (цены после повышения на 20%). Каждую цену выводить в виде: `Новая цена: XXX руб.`

---

**Упражнение 3.** Переделайте код из упражнения 2, используя `Продолжить` вместо вложенного `Если`. Задача: вывести только те *повышенные* цены, которые превышают 50 рублей.

---

**Упражнение 4.** В «Канцтоварах» нужно сформировать новый массив — только цены после скидки 15%. Скидка применяется только к товарам дороже 20 рублей. Напишите программу, которая создаёт этот массив и выводит его содержимое. Данные: `[35, 60, 15, 25, 10]`.

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

Это **перебор с фильтрацией**. Цикл проходит всю коллекцию, не останавливаясь. `Прервать` отсутствует. Условие внутри цикла — фильтр: выводятся только товары с длиной названия больше 5 символов. Нет переменной-результата «найден/не найден».

---

**К упражнению 2.** *Полное решение.*

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);
Цены.Добавить(10);

Для каждого Цена Из Цены Цикл
    НоваяЦена = Цена * 1.2;
    Сообщить("Новая цена: " + НоваяЦена + " руб.");
КонецЦикла;
```

Экран: `Новая цена: 42 руб.`, `Новая цена: 72 руб.`, `Новая цена: 18 руб.`, `Новая цена: 30 руб.`, `Новая цена: 12 руб.`.

---

**К упражнению 3.** *Указание.*

Нужно вывести повышенную цену только если она > 50. С `Продолжить`: внутри цикла первым делом проверяем `Если НоваяЦена <= 50 Тогда Продолжить; КонецЕсли;` — и дальше код вывода выполнится только для прошедших проверку.

---

**К упражнению 4.** *Полное решение.*

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);
Цены.Добавить(10);

ЦеныСоСкидкой = Новый Массив;
Для каждого Цена Из Цены Цикл
    Если Цена > 20 Тогда
        ЦеныСоСкидкой.Добавить(Цена * 0.85);
    Иначе
        ЦеныСоСкидкой.Добавить(Цена); // скидка не применяется
    КонецЕсли;
КонецЦикла;

Для каждого Цена Из ЦеныСоСкидкой Цикл
    Сообщить(Цена + " руб.");
КонецЦикла;
```

Экран: `29.75 руб.`, `51 руб.`, `15 руб.`, `21.25 руб.`, `10 руб.`.

Товары по 15 и 10 рублей — дешевле 20 рублей, скидка не применяется, цена сохраняется без изменений.
