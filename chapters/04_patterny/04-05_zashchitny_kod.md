## § 4.5. Проверки и защитный код

В предыдущих параграфах мы строили паттерны на «хороших» данных: коллекции были непустыми, поиск находил то, что искали, деление всегда было на ненулевой знаменатель. Реальные данные так не ведут себя. Коллекция может оказаться пустой. Поиск — ничего не найти. Пользователь — ввести что-то неожиданное.

Программа, которая не готова к таким случаям, ведёт себя непредсказуемо: выдаёт неверный результат или останавливается с ошибкой. Чтобы этого не происходило, пишут **защитный код** — проверки, которые выявляют проблему до того, как она стала катастрофой.

---

## 1. Что такое защитный код

Защитный код — это проверка условия перед тем, как выполнить действие, которое может дать неверный результат при «плохих» данных.

Два ключевых вопроса:
- *Что может пойти не так?*
- *Как мне это проверить заранее?*

> **Защитный код: проверить данные перед работой с ними. Лучше явно обработать нештатный случай, чем получить молчаливо неверный результат.**

Это не параноя — это профессиональный рефлекс. Хорошая программа знает о своих допущениях и проверяет их.

---

## 2. Проверка пустой коллекции

В § 4.4 мы посчитали среднее и добавили проверку `Если Количество > 0`. Это и есть защита от деления на ноль. Сделаем то же самое системно.

Перед любым кодом, который предполагает непустую коллекцию, стоит проверить:

```
Если Коллекция.Количество() = 0 Тогда
    Сообщить("Список пуст — нечего обрабатывать");
Иначе
    // основная работа
КонецЕсли;
```

Или в виде «раннего выхода» из процедуры/функции — через `Возврат`:

```
Функция НайтиМаксимум(Цены)
    Если Цены.Количество() = 0 Тогда
        Возврат 0; // или другое значение-заглушка
    КонецЕсли;

    МаксимумНайден = Ложь;
    Максимум = 0;
    Для каждого Цена Из Цены Цикл
        Если НЕ МаксимумНайден Тогда
            Максимум = Цена;
            МаксимумНайден = Истина;
        ИначеЕсли Цена > Максимум Тогда
            Максимум = Цена;
        КонецЕсли;
    КонецЦикла;

    Возврат Максимум;
КонецФункции;
```

Ранний `Возврат` в начале функции — это **сторожевая проверка** (guard clause): она немедленно обрабатывает нештатный случай и возвращает управление, не загромождая основную логику вложениями.

> **Сторожевая проверка: вынести нештатный случай в начало подпрограммы и выйти из неё немедленно. Основной код остаётся плоским и читаемым.**

Метод `.Количество()` здесь стоит мимоходом — это вызов метода объекта «Массив». Методы объектов — тема § 5.1; пока воспринимайте `.Количество()` как встроенный способ узнать размер коллекции.

---

## 3. Проверка результата поиска

После паттерна поиска переменная-результат содержит `Неопределено`, если элемент не найден. Работать с `Неопределено` как с обычным значением — ошибка: сложить его с числом или склеить со строкой не получится. Поэтому перед использованием результата — проверка:

```
НайденныйТовар = Неопределено;
Для каждого Товар Из Товары Цикл
    Если Товар = "Ластик" Тогда
        НайденныйТовар = Товар;
        Прервать;
    КонецЕсли;
КонецЦикла;

// НЕЛЬЗЯ сразу делать:
// Сообщить("Найден: " + НайденныйТовар);
// Потому что НайденныйТовар может быть Неопределено

// НУЖНО сначала проверить:
Если НайденныйТовар = Неопределено Тогда
    Сообщить("Товар не найден");
Иначе
    Сообщить("Найден: " + НайденныйТовар);
КонецЕсли;
```

Это не опция — это обязательная часть паттерна поиска. Поиск без проверки результата — незавершённый паттерн.

---

## 4. Защита от деления на ноль

Деление — частый источник ошибок. Перед каждым делением, где знаменатель вычисляется (а не является константой), нужна проверка:

```
Если Количество <> 0 Тогда
    Среднее = Сумма / Количество;
    Сообщить("Среднее: " + Среднее);
Иначе
    Сообщить("Нет данных для расчёта среднего");
КонецЕсли;
```

Правило простое: делитель — переменная? Проверь, что она не ноль.

---

## 5. Типичные места для защитного кода

Защитный код появляется не везде — только там, где есть реальный риск получить «плохие» данные. Типичные места:

```
  Ситуация                           Защита
  ──────────────────────────────     ────────────────────────────────
  работа с коллекцией                проверить, что не пуста
  деление на вычисленный знаменатель проверить, что знаменатель ≠ 0
  использование результата поиска    проверить, что не Неопределено
  работа с входными данными функции  проверить корректность параметров
```

---

## 6. Защитный код и читаемость

Защитный код имеет репутацию «раздувающего» программу: вместо трёх строк логики — десять со всеми проверками. Это верно, но и цена молчаливой ошибки выше.

Два приёма, которые не дают защитному коду перегружать основную логику:

**Ранний выход** уже описан выше: нештатные случаи обрабатываются в начале функции, основной код не вкладывается в `Иначе`.

**Вынесение проверок в отдельную функцию:**

```
Функция КоллекцияНеПуста(Коллекция)
    Возврат Коллекция.Количество() > 0;
КонецФункции;

// Использование
Если НЕ КоллекцияНеПуста(Цены) Тогда
    Сообщить("Нет данных");
Иначе
    // основная логика
КонецЕсли;
```

Имя функции `КоллекцияНеПуста` читается как условие — это приём именованного условия из § 3.1.

## 7. Канцтовары: надёжный финансовый отчёт

Возьмём программу из § 4.4 и добавим защитный код:

```
Цены = Новый Массив;
// В этот раз намеренно не добавляем данные — проверяем защиту

Если Цены.Количество() = 0 Тогда
    Сообщить("Нет товаров для отчёта");
Иначе
    Итого = 0;
    МаксимумНайден = Ложь;
    МаксимальнаяЦена = 0;

    Для каждого Цена Из Цены Цикл
        Итого = Итого + Цена;
        Если НЕ МаксимумНайден Тогда
            МаксимальнаяЦена = Цена;
            МаксимумНайден = Истина;
        ИначеЕсли Цена > МаксимальнаяЦена Тогда
            МаксимальнаяЦена = Цена;
        КонецЕсли;
    КонецЦикла;

    Среднее = Итого / Цены.Количество(); // безопасно: мы уже знаем, что > 0
    Сообщить("Итого: " + Итого + " руб.");
    Сообщить("Максимум: " + МаксимальнаяЦена + " руб.");
    Сообщить("Среднее: " + Среднее + " руб.");
КонецЕсли;
```

При пустом массиве программа выведет: `Нет товаров для отчёта` — и не упадёт.

Обратите внимание: деление `Итого / Цены.Количество()` безопасно, потому что мы уже знаем из условия `Если Цены.Количество() = 0`, что за `Иначе` коллекция непуста. Это называют *инвариантом*: к этой точке кода мы знаем нечто гарантированное.

---

## Главное в одном абзаце

Защитный код — это проверки, которые обрабатывают нештатные данные до того, как они вызовут неверный результат или ошибку. Три самых частых места: пустая коллекция (проверить `.Количество()` перед циклом), результат поиска (проверить на `Неопределено` перед использованием), деление (проверить знаменатель на ноль). Сторожевая проверка выносит нештатный случай в начало подпрограммы и немедленно возвращает управление через `Возврат` — основная логика остаётся плоской. После блока защиты программа работает в условиях, которые она сама проверила: это инвариант, на который можно опираться.

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

```
Функция СреднееЗначение(Числа)
    Если Числа.Количество() = 0 Тогда
        Возврат 0; // защита: пустой массив → среднее = 0
    КонецЕсли;

    Сумма = 0;
    Для каждого Число Из Числа Цикл
        Сумма = Сумма + Число;
    КонецЦикла;
    Возврат Сумма / Числа.Количество();
КонецФункции;
```

Сторожевая проверка в самом начале: если пусто — немедленно вернуть 0. Деление в последней строке безопасно — мы знаем, что `Количество() > 0`.

---

**К упражнению 2.** *Полное решение.*

Одно место: строка `Скидка = НайденнаяЦена * 0.1`.

После внутреннего цикла `НайденнаяЦена` может быть `Неопределено` (если в массиве `Цены` нет ни одного положительного числа). Умножение `Неопределено * 0.1` приведёт к ошибке.

Нужна проверка перед использованием:

```
Если НайденнаяЦена = Неопределено Тогда
    Сообщить(Товар + ": цена не найдена, скидка не рассчитана");
Иначе
    Скидка = НайденнаяЦена * 0.1;
    Сообщить(Товар + ": скидка " + Скидка + " руб.");
КонецЕсли;
```

---

**К упражнению 3.** *Полное решение.*

```
Процедура ПоказатьМаксимум(Цены)
    Если Цены.Количество() = 0 Тогда
        Сообщить("Нет данных");
        Возврат;
    КонецЕсли;

    МаксимумНайден = Ложь;
    Максимум = 0;
    Для каждого Цена Из Цены Цикл
        Если НЕ МаксимумНайден Тогда
            Максимум = Цена;
            МаксимумНайден = Истина;
        ИначеЕсли Цена > Максимум Тогда
            Максимум = Цена;
        КонецЕсли;
    КонецЦикла;

    Сообщить("Максимум: " + Максимум);
КонецПроцедуры;
```

Обратите внимание: подпрограмма стала `Процедурой`, а не функцией — она ничего не возвращает, только выводит. Сторожевая проверка — в начале. После неё основной код не вложен ни в какой `Иначе`.
