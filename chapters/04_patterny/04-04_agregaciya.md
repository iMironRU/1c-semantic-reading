## § 4.4. Агрегация: сумма, максимум, счётчик

Перебор (§ 4.3) обходит коллекцию и обрабатывает каждый элемент *отдельно* — выводит, преобразует, сохраняет. Агрегация делает другое: она обходит коллекцию, чтобы *свернуть* её к одному результату. Не «напечатать каждую цену», а «подсчитать общую сумму». Не «вывести каждый товар», а «найти самый дорогой».

Результат агрегации — одно значение, которое описывает коллекцию целиком.

---

## 1. Типичная форма паттерна

Все варианты агрегации устроены по одной схеме:

```
Результат = <начальное значение>;
Для каждого Элемент Из Коллекция Цикл
    Результат = <обновление Результата через Элемент>;
КонецЦикла;
// Результат содержит агрегат всей коллекции
```

Три части:
1. **Инициализация:** переменная результата получает стартовое значение.
2. **Цикл:** каждый элемент «вносит вклад» в результат.
3. **Итог:** после цикла переменная хранит агрегат.

> **Паттерн агрегации: инициализировать результат, пройти по коллекции и на каждом шаге обновить результат с учётом текущего элемента.**

Разница между вариантами агрегации — в том, каким именно образом элемент «вносит вклад». Разберём три самых частых.

---

## 2. Сумма

Задача: сложить все элементы коллекции.

```
Итого = 0;
Для каждого Цена Из Цены Цикл
    Итого = Итого + Цена;
КонецЦикла;
```

Начальное значение — `0`: ноль — нейтральный элемент для сложения (добавление нуля ничего не меняет). На каждом шаге текущая сумма увеличивается на очередной элемент.

Прогонка для `[35, 60, 15]`:

| Шаг | Цена | Итого до | Итого после |
|-----|------|----------|-------------|
| 1   | 35   | 0        | 35          |
| 2   | 60   | 35       | 95          |
| 3   | 15   | 95       | 110         |

Результат: 110.

---

## 3. Максимум

Задача: найти наибольший элемент коллекции.

Здесь начальное значение — не нейтральный ноль. Если начать с нуля и все элементы окажутся отрицательными — результат будет неверным. Правильный старт — *первый элемент коллекции*.

Поскольку мы не можем обратиться к первому элементу напрямую без числового индекса (это возможность, которую разберём в § 5.1), используем флаг:

```
МаксимумНайден = Ложь;
Максимум = 0; // значение неважно до первой итерации

Для каждого Элемент Из Коллекция Цикл
    Если НЕ МаксимумНайден Тогда
        Максимум = Элемент;      // первый элемент — стартовое значение
        МаксимумНайден = Истина;
    ИначеЕсли Элемент > Максимум Тогда
        Максимум = Элемент;
    КонецЕсли;
КонецЦикла;
```

Первая итерация всегда инициализирует `Максимум` реальным элементом. Все последующие — сравнивают с накопленным максимумом.

Прогонка для `[35, 60, 15]`:

| Шаг | Элемент | МаксимумНайден | Максимум |
|-----|---------|----------------|----------|
| 1   | 35      | Ложь → Истина  | 35       |
| 2   | 60      | Истина; 60 > 35| 60       |
| 3   | 15      | Истина; 15 > 60 — нет | 60 |

Результат: 60.

> **Для максимума начальное значение нельзя брать произвольно — оно должно быть первым элементом коллекции. Флаг первой итерации позволяет инициализировать правильно.**

*Примечание:* если коллекция заведомо содержит только положительные числа, можно упростить и начать с нуля. Но это предположение стоит делать осознанно — и лучше его где-то зафиксировать комментарием.

---

## 4. Счётчик

Задача: подсчитать количество элементов, удовлетворяющих условию.

```
Счётчик = 0;
Для каждого Элемент Из Коллекция Цикл
    Если <условие на Элемент> Тогда
        Счётчик = Счётчик + 1;
    КонецЕсли;
КонецЦикла;
```

Начальное значение — `0`. На каждом шаге, если элемент подходит, счётчик увеличивается на 1.

Пример: посчитать, сколько цен превышают 20 рублей.

```
Счётчик = 0;
Для каждого Цена Из Цены Цикл
    Если Цена > 20 Тогда
        Счётчик = Счётчик + 1;
    КонецЕсли;
КонецЦикла;
Сообщить("Дорогих товаров: " + Счётчик);
```

Прогонка для `[35, 60, 15, 25, 10]`:

| Шаг | Цена | Условие `> 20` | Счётчик |
|-----|------|----------------|---------|
| 1   | 35   | Истина         | 1       |
| 2   | 60   | Истина         | 2       |
| 3   | 15   | Ложь           | 2       |
| 4   | 25   | Истина         | 3       |
| 5   | 10   | Ложь           | 3       |

Экран: `Дорогих товаров: 3`.

---

## 5. Агрегация нескольких значений одновременно

Ничто не мешает вычислять несколько агрегатов за один проход. Сумма и счётчик вместе — это среднее:

```
Сумма = 0;
Количество = 0;

Для каждого Цена Из Цены Цикл
    Сумма = Сумма + Цена;
    Количество = Количество + 1;
КонецЦикла;

Если Количество > 0 Тогда
    Среднее = Сумма / Количество;
    Сообщить("Средняя цена: " + Среднее + " руб.");
КонецЕсли;
```

Обратите внимание на проверку `Количество > 0` перед делением: делить на ноль нельзя. Это уже элемент защитного кода — паттерна, который мы разберём в § 4.5.

---

## 6. Агрегации в одном ряду: сравнение

Три паттерна агрегации выглядят похоже, но каждый решает свою задачу:

```
  Задача       Инициализация   Обновление в цикле
  ──────────   ──────────────  ──────────────────────────────
  Сумма        0               Результат = Результат + Элемент
  Максимум     первый элемент  Если Элемент > Результат → обновить
  Счётчик      0               Результат = Результат + 1 (если условие)
```

Все три — варианты одного паттерна с разным «весом» вклада каждого элемента.

---

## 7. Канцтовары: финансовый отчёт

Программа считает итоговую сумму заказа, максимальную цену и количество дорогих позиций:

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);
Цены.Добавить(10);

// Три агрегата за один проход
Итого = 0;
МаксимумНайден = Ложь;
МаксимальнаяЦена = 0;
КоличествоДорогих = 0;

Для каждого Цена Из Цены Цикл
    Итого = Итого + Цена;

    Если НЕ МаксимумНайден Тогда
        МаксимальнаяЦена = Цена;
        МаксимумНайден = Истина;
    ИначеЕсли Цена > МаксимальнаяЦена Тогда
        МаксимальнаяЦена = Цена;
    КонецЕсли;

    Если Цена > 30 Тогда
        КоличествоДорогих = КоличествоДорогих + 1;
    КонецЕсли;
КонецЦикла;

Сообщить("Итого: " + Итого + " руб.");
Сообщить("Максимальная цена: " + МаксимальнаяЦена + " руб.");
Сообщить("Позиций дороже 30 руб.: " + КоличествоДорогих);
```

Экран:

```
Итого: 145 руб.
Максимальная цена: 60 руб.
Позиций дороже 30 руб.: 2
```

Три агрегата — один цикл. Это типичный приём: если нужно несколько агрегатов, лучше обойти коллекцию один раз, чем три раза по отдельности.

---

## Главное в одном абзаце

Паттерн агрегации сворачивает коллекцию к одному результату: инициализируем переменную-накопитель перед циклом, на каждой итерации обновляем её с учётом текущего элемента, после цикла в ней хранится агрегат всей коллекции. Три самых частых варианта: сумма (начало с 0, обновление через сложение), максимум (начало с первого элемента через флаг, обновление при превышении), счётчик (начало с 0, обновление при выполнении условия). Несколько агрегатов можно вычислять за один проход, объединяя переменные и их обновления в одном цикле. Деление (для среднего) требует предварительной проверки, что знаменатель не равен нулю — это уже защитный код.

**Упражнение 1.** Напишите программу, которая находит *минимальную* цену в массиве `[35, 60, 15, 25, 10]`. Используйте флаговый подход, как в примере с максимумом.

**Упражнение 2.** Прогоните программу вычисления суммы для пустого массива (нет добавленных элементов). Что окажется в переменной `Итого` после цикла? Почему?

---

**Упражнение 3.** Найдите ошибку:

```
Максимум = 0;
Для каждого Цена Из Цены Цикл
    Если Цена > Максимум Тогда
        Максимум = Цена;
    КонецЕсли;
КонецЦикла;
Сообщить("Максимум: " + Максимум);
```

Для каких данных эта программа даст неверный результат?

---

**Упражнение 4.** В «Канцтоварах» нужно узнать: сколько товаров в списке, какова общая стоимость и средняя цена. Напишите программу. Данные: `[35, 60, 15, 25, 10]`.

---

## Ответы, указания, разборы

**К упражнению 1.** *Полное решение.*

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);
Цены.Добавить(10);

МинимумНайден = Ложь;
Минимум = 0;

Для каждого Цена Из Цены Цикл
    Если НЕ МинимумНайден Тогда
        Минимум = Цена;
        МинимумНайден = Истина;
    ИначеЕсли Цена < Минимум Тогда
        Минимум = Цена;
    КонецЕсли;
КонецЦикла;

Сообщить("Минимальная цена: " + Минимум + " руб.");
```

Прогонка: первый элемент (35) → `Минимум = 35`. Второй (60): 60 < 35 — нет. Третий (15): 15 < 35 — да, `Минимум = 15`. Четвёртый (25): 25 < 15 — нет. Пятый (10): 10 < 15 — да, `Минимум = 10`.

Результат: `10 руб.`

---

**К упражнению 2.** *Полное решение.*

Для пустого массива цикл не выполнится ни разу — нет ни одного элемента. Переменная `Итого` останется равной своему начальному значению: `0`.

Это корректный результат: сумма пустого набора равна нулю. Паттерн суммы работает правильно даже для пустой коллекции, потому что `0` — нейтральный элемент для сложения.

---

**К упражнению 3.** *Полное решение.*

Ошибка: `Максимум = 0` как начальное значение. Если все цены отрицательные — например, `[-5, -3, -10]` — ни одна не превысит ноль, и `Максимум` останется `0`, хотя правильный ответ — `-3`.

Для данных `[35, 60, 15, 25, 10]` — всё положительно, программа работает верно. Ошибка проявляется только при отрицательных или нулевых значениях.

Исправление — флаговый подход (как в разделе 3) или явная проверка на пустую коллекцию с инициализацией первым элементом.

---

**К упражнению 4.** *Полное решение.*

```
Цены = Новый Массив;
Цены.Добавить(35);
Цены.Добавить(60);
Цены.Добавить(15);
Цены.Добавить(25);
Цены.Добавить(10);

Количество = 0;
Итого = 0;

Для каждого Цена Из Цены Цикл
    Количество = Количество + 1;
    Итого = Итого + Цена;
КонецЦикла;

Сообщить("Количество товаров: " + Количество);
Сообщить("Общая стоимость: " + Итого + " руб.");

Если Количество > 0 Тогда
    Среднее = Итого / Количество;
    Сообщить("Средняя цена: " + Среднее + " руб.");
КонецЕсли;
```

Экран:
```
Количество товаров: 5
Общая стоимость: 145 руб.
Средняя цена: 29 руб.
```

Проверка `Количество > 0` перед делением защищает от деления на ноль при пустом массиве.
